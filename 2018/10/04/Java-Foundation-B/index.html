位运算符

System.out.println(5 & 9);       //输出：1
System.out.println(5 | 9);       //输出：13
System.out.println(~ -5);        //输出：4
System.out.println(5 ^ 9);       //输出：12
System.out.println(-5 << 2);     //输出：-20
System.out.println(-5 >> 2);     //输出：-2
System.out.println(-5 >>> 2);    //输出：1073741822
  • &
    • 按位与
    • 两位同时为1，才返回1
  • |
    • 按位或
    • 有位个为1，就返回1
  • ~
    • 按位非
    • 单目运算符
    • 将操作数的每个位（包括符号位）全部取反
  • ^
    • 按位异或
    • 两位不同，返回1
    • 两位相同，返回0
  • <<
    • 左移运算符
    • 右边空出来的以0补充
  • >>
    • 右移运算符
    • 左边空出来的以符号位数字补充
  • >>>
    • 无符号右移运算符
    • 左边空出来的以0补充
  • 对于 byte 、short 、char，先自动升级为 int 类型后再移位
  • int 类型：a >> b
    • b > 32
    • 系统先用 b 对32求余
    • 得到结果为移位数
    • a >> 33 = a >> 1
  • long 类型：a >> b
    • b > 64
    • 系统先用 b 对64求余
    • 得到的结果为移位数
    • a >> 65 = a >> 1
  • 进行移位运算不会改变操作数本身
    • 只是得到了一个移位运算后的值
    • 原来的操作数本身不会发生改变

三目运算符

int    e1 = 11;
double e2 = 11.0;

System.out.println(e1 > e2 ? "e1 大于 e2" :(e1 < e2 ? "e1 小于 e2" : "e1 等于 e2"));
/*
输出：e1 = e2
*/
  • (expression) ? if-true-statement : if-false-statement
    • 先对逻辑表达式 expression 求值
      • 返回 true
        • 则返回第二个操作数的值
      • 返回 false
        • 则返回第三个操作数的值
  • 三目运算符是 if else 的精简写法
    • 三目运算符不支持多个语句
    • if else 后的代码块可以有多个语句
  • 三目运算符可以嵌套
    • 以处理更复杂的情况

运算符的结合性和优先级

  • 结合性
    • Java 中大部分运算符是从左向右结合
    • 从右向左结合
      • 单目运算符
      • 赋值运算符
      • 三目运算符
        • a?b:c?d:e?f:g = a?b:(c?d:(e?f:g)).
    • + 和 - 是两个可结合的运算
      • 互换位置不会影响结果
  • 优先级
    • 源代码就是一份文档：
      • 源代码的可读性比运行效率更重要
      • 不要把一个表达式写得过于复杂
      • 不要过多地依赖运算符的优先级来控制表达式的执行顺序
      • 尽量使用 ()

流程控制结构

简介

  • 1966年，计算机科学家 Bohm 和 Jacopini 证明了任何简单或复杂的算法都可以由下列三种基本结构组合而成：
    • 顺序结构：按照源代码中的排列顺序从上到下依次执行
    • 选择结构：单选，双选，多选
    • 循环结构：当型（先判断后执行），直到型（先执行后判断）

选择结构

int a = 5;

if(a > 4)
    a--;                            //if 语句唯一的条件执行体
    System.out.println("a 大于4");   //已经不属于 if 语句的条件执行体
else                                //没有 if 语句对应，从而引起编译错误
    System.out.println("a 小于4");
  • 若没有 { } 则 if 条件语句只控制到紧跟该条件语句的第一个分号处
  • 即使只有一行代码，也推荐加上花括号 { } 
int age1 = 45;

if(age1 > 20)
{
    System.out.println("青年人");
}    

else if(age1 > 40)
{
    System.out.println("中年人");
}    

else if(age1 > 60)
{
    System.out.println("老年人");
}                                      //最后的 else 也可以省略
/*
输出：青年人
实际上希望判断45岁是中年人
表面上看起来 else 后面没有条件
或者 else if 后面只有一个条件
但真相是
else 本身就是一个条件
else 隐含的条件就是对前面条件的取反
所以上面的三个条件实际上对应
    1.age1 > 20  =  age1 > 20
    2.age1 > 40  =  age1 > 40 && !(age1 > 20)
    3.age1 > 60  =  age1 > 60 && !(age1 > 40) && !(age1 > 40 && !(age1 > 20))
条件2和条件3永远不可能发生
*/

int age2 = 45;

if(age2 > 60)
{
    System.out.println("青年人");
}    

else if(age2 > 40)
{
    System.out.println("中年人");
}    

else if(age2 > 20)
{
    System.out.println("老年人");
}     
/*
输出：中年人
上面的三个条件实际上对应

      -年龄大于40，且小于60

*/

    • 总是优先把包含范围小的条件放在前面处理
import java.lang.String;

String season = "夏天";
switch (season)
{
    case "春天":
         System.out.println("春暖花开");
         break;
    case "夏天":
         System.out.println("夏日炎炎");
         break;
    case "秋天":
         System.out.println("秋高气爽");
         break;
    case "冬天":
         System.out.println("冬雪皑皑");
         break;
    default:
         System.out.println("季节输入错误");
}
/*
输出：夏日炎炎
*/

    • 控制表达式的数据类型只能是
      • byte 、short 、char 、int
      • 枚举 enum
      • java.lang.String
        • Java 7 以上版本
    • 可以省略 case 后面的 { }
    • default 和 else 类似
      • 看似没有条件其实有隐含条件
      • 即 switch 括号内表达式的值不能与前面任何一个 case 标签后的值相等
    • 没有 break;
      • 则会从匹配到的第一个 case 开始一路向下执行
      • 不再判断 case 和 default
      • 直到遇到 break 结束

循环结构

int count = 1;

while(count < 10)
{
    System.out.println(count);
    count++;
}

System.out.println("循环结束！");
/*
输出：
1 ~ 9
循环结束！
*/
  • while
    • 先判断，后执行
    • 如果省略了循环体的花括号 { }
    • 那么 while 循环条件仅仅控制到跟进该循环条件的第一个分号处
    • 例如循环体部分和迭代语句合并在一起
    • 但通常建议加上花括号
    • 两个陷阱
      • 死循环
        • 循环条件一直为 true
      • while 循环条件后紧跟一个分号 ;
        • 一个单独的分号 ; 表示一个空语句
        • 不做任何事的空语句
        • 这意味着循环体是空语句
        • 同时循环条件的返回值没有任何改变
        • 也成了一个死循环
int count = 1;

do
{

    count++;
}while(count < 10);

System.out.println("循环结束！");
/*
输出：
1 ~ 9
循环结束！
*/
  • do while
    • 先执行，后判断
    • do while 循环至少执行一次
    • do while 的循环条件后必须有一个 ;
      • 表明循环结束
    • 循环体部分和迭代语句合并在一起，可以省略 { }
      • 但通常建议加上花括号
for(int count = 1 ; count < 10 ; count++)
{
    System.out.println(count);

}

System.out.println("循环结束！");
/*
输出：
1 ~ 9
循环结束！
*/

for(int incount = 1 ; incount < 10 ; incount++)
{
    System.out.println(incount);
    outcount = incount;
}

System.out.println("循环结束！");

  • for
    • 更加简洁
    • 可以代替 while 、do while 循环语句
    • 循环条件总比循环体要多执行一次
    • 循环迭代语句与循环体分离
      • 可无视 continue 的作用
    • 如果循环体只有一行，可省略花括号 { }
      • 但通常建议加上花括号
    • 初始化语句
      • 单个初始化
      • 多个初始化
        • 必须是相同的数据类型，即只能有一个声明语句
    • 循环条件
      • boolean 表达式
      • 包含逻辑运算符的表达式
    • 建议不要在循环体内修改循环变量（即循环计数器）的值
    • 初始化语句、循环条件、迭代语句
      • 都可以省略
      • 省略循环条件，循环条件默认为 true，会变成一个死循环
    • for 循环里定义的变量
      • 作用域仅在循环体内有效
      • for 循环结束后，这些变量将不可访问

数组

理解数组

  • 一个数组中，数组元素的类型是唯一的
  • 一旦数组初始化完成，数组在内存中所占的空间将被固定下来，数组的长度不可改变
  • 即使把数组元素的数据清空，其所占空间依然被保留，长度依然不变
  • 数组既可以存储基本类型的数据，也可以存储引用类型的数据
  • 数组本身也是一种引用类型

定义和初始化数组

  • 定义数组的两种方式

    • type[] arrayName
    • type arrayName[]
    • 推荐使用第一种方式，更加直观清晰
      • arrayName 是变量名
      • type[] 是变量类型

  • 定义数组时不能指定数组的长度，因为这个引用类型的变量还未指向任何有效的内存

  • Java 中的数组必须先初始化，才能使用

    • 即为数组的数组元素分配内存空间

    • 并必须为每个数组元素赋初始值

    • 静态初始化

      • 由程序员显式分配每个数组元素的初始值，由系统决定数组的长度

        `java
        int[] a;
        a = new int[]{5 , 6 , 8 , 20};

String[] poems = {
                   "红豆生南国",
                   "春来发几枝",
                   "愿君多采撷",
                   "此物最相思",
                 };         

for (String poem : poems)           //形参 poem 将自动迭代遍历每个数组元素

     poem = "红豆生南国";
     System.out.println(poem);    
}
/*
输出：
红豆生南国
红豆生南国
红豆生南国
红豆生南国
给循环变量 poem 赋值虽然语法上可行
但没有意义
且会引起错误
循环变量 poem 相当于一个临时变量
系统会把数组元素依次赋给这个临时变量
这个临时变量不是数组元素
它只是保存了数组元素的值
如果希望改变数组元素的值
则不应该用 foreach 循环
*/
  • foreach
  • 无需获得数组（或集合）的长度即可遍历数组（或集合）
  • 无需根据索引来访问数组元素（或集合元素）
  • 无需循环条件、无需迭代语句
  • 由系统自动完成
  • 循环过程中不会改变数组元素的值
    • 不要对 foreach 的循环变量赋值
