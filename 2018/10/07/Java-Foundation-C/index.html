总结

  • 成员变量 + 方法 = 类定义
  • 状态数据 + 行为 = 对象

定义类

[修饰符]   class  类名
{
    0 ~ n 个成员变量......
    0 ~ n 个方法......
    0 ~ n 个构造器......
}
  • 修饰符
    • public 、final 、abstract
    • 或省略
  • 类名
    • 一个合法的标识符
    • 由有意义的英文单词连缀而成（不得有分隔符），每个单词首字母大写，其他皆小写（可读性）
  • 类的三个基本成员
    • 成员变量
    • 方法
    • 构造器
  • static
    • static 修饰的成员表明其属于这个类本身，而不属于这个类的单个实例
      • 类变量 （静态变量）
      • 类方法 （静态方法）
    • 没有 static 修饰的成员属于这个类的单个实例，而不属于这个类
      • 实例变量 （非静态变量）
      • 实例方法 （非静态方法）
    • static 成员不能直接访问非 static 成员
    • 静态成员不能直接访问非静态成员

成员变量

[修饰符]  类型  成员变量名  [= 默认值]
  • 修饰符
    • public 、protected 、private （最多只能出现其一）
    • final 、static （可与上面组合）
    • 或省略
  • 类型
    • 基本类型
    • 引用类型
  • 成员变量名
    • 一个合法的标识符
    • 由有意义的英文单词连缀而成（不得有分隔符），第一个单词首字母小写，后面每个单词首字母大写，其他皆小写（可读性）
  • 可以指定默认值
class Poem
{
      public String s1;

      public static String s2;
}

public class Dog
{ 
       public static void main(String[] args)
       {
              System.out.println(Poem.s2);
              /*
              类自动初始化
              输出：
              null
              */

              Poem p = new Poem();        //创建 Poem 类的实例对象并赋给 Poem 类型变量 p 

              System.out.println(p.s1 + '\n' + p.s2);
              /*
              通过 Poem 对象的引用 P 访问实例变量 s1 和类变量 s2
              输出：
              null
              null
              */

              p.s1 = "往事只堪哀，对景难排";            //直接为实例变量 S1 赋值

              p.s2 = "一任珠帘闲不卷，终日谁来？";      //通过 P 访问类变量 s2 

              System.out.println(p.s1 + '\n' + p.s2);
              /*
              再次通过 Poem 对象的引用 P 访问实例变量 s1 和类变量 s2
              输出：
              往事只堪哀，对景难排
              一任珠帘闲不卷，终日谁来？
              */

              System.out.println(Poem.s2);
              /*
              通过类本身访问类变量 s2
              输出：
              一任珠帘闲不卷，终日谁来？
              */

               Poem t = new Poem();        //创建 Poem 类的实例对象并赋给 Poem 类型变量 t

              System.out.println(t.s1 + '\n' + t.s2);
              /*
              通过 Poem 对象的引用 t 访问实例变量 s1 和类变量 s2
              输出：
              null
              一任珠帘闲不卷，终日谁来？
              */

               Poem.s2 = "相得玉楼瑶殿影，空照秦淮。";      //通过类 Poem 访问类变量 s2 并赋值

              Poem f = new Poem();        //创建 Poem 类的实例对象并赋给 Poem 类型变量 f

              System.out.println(f.s1 + '\n' + f.s2);
              /*
              通过 Poem 对象的引用 f 访问实例变量 s1 和类变量 s2
              输出：
              null
              相得玉楼瑶殿影，空照秦淮。
              */
       }
}
  • 成员变量（类变量 + 实例变量）
    • 类变量作为类本身的一部分，与类本身共存亡
    • 实例变量作为实例的一部分，与实例共存亡
    • 类变量的作用域大于实例变量的作用域
  • 如果通过一个实例修改了类变量的值，修改的依然是这个类的类变量，与通过类来修改变量的结果完全相同，这会导致该类的其他实例来访问这个类变量时，也会获得这个被修改过的值
    • 同一个类所有的实例对象访问类变量时，实际上访问的是该类本身的同一个变量
      • 也就是说，访问了同一片内存区
  • 只要为一个类定义了类变量或实例变量
    • 系统就会自动在这个类的准备阶段或创建该类的实例对象时，自动进行默认初始化
    • 与数组动态初始化时的赋值规则完全相同
  • 同一类里，不能定义两个同名的成员变量，即使一个是类变量，一个是实例变量，也不行
public class HelloWorld
{
       private String s1 = "醉乡路稳宜频到" + '\n' + "此外不堪行";         //定义实例变量 s1

       private static String s2 = "烛残漏滴频欹枕" + '\n' + "起坐不能平";  //定义类变量 s2

       public static void main(String[] args)
     {
              String s2 = "昨夜风兼雨" + '\n' + "帘帏飒飒秋声";       //定义方法局部变量 s2

              System.out.println(s2);                               //输出方法局部变量 s2

              System.out.println(HelloWorld.s2);                    //输出类变量 s2

              new HelloWorld().info();           //创建 HelloWorld 类的实例对象调用实例方法 info

              /*
              昨夜风兼雨
               帘帏飒飒秋声
               烛残漏滴频欹枕
               起坐不能平
               世事漫随流水
               算来一梦浮生
               醉乡路稳宜频到
               此外不堪行
              */
     }     

       public void info()
     {
              String s1 = "世事漫随流水" + '\n' + "算来一梦浮生";     //定义方法局部变量 s1

              System.out.println(s1);                              //输出方法局部变量 s1

              System.out.println(this.s1);                         //输出实例变量 s1
     }
}
  • 局部变量（形参 + 方法局部变量 + 代码块局部变量）
    • 形参的作用域在整个方法内有效
    • 方法局部变量的作用域从定义开始，到方法结束时失效
    • 代码块局部变量的作用域从定义开始，到代码块结束时失效
  • 除了形参，方法局部变量和代码块局部变量必须显式初始化，才能访问它们
    • 定义时，系统并未为它们分配内存空间
    • 直到为它们赋初始值时，系统才会为它们分配可访问的内存
  • 一个方法内不能定义两个同名的方法局部变量
  • 方法局部变量不能与形参同名
  • 同一个方法中不同的代码块内的代码块局部变量可以同名
  • 如果先定义代码块局部变量，后定义方法局部变量
    • 前面的代码块局部变量和后面的方法局部变量也可以同名
  • Java 允许局部变量和成员变量同名
    • 如果是在方法内同名
    • 局部变量会覆盖成员变量
    • 如果在方法里需要引用成员变量
      • 可使用 this（实例变量） 或类名（类变量）作为主调
  • 不过大部分时候还是应该尽量避免这种同名现象

第三个程序才最符合软件开发规范：一个循环变量只需要在循环体内有效

  • 定义一个成员变量（类变量 + 实例变量）时，成员变量将会被放置在堆内存中
  • 成员变量的作用域将扩大到类存在范围或对象存在范围
    • 增大了变量的生存时间，将导致更大的内存开销
    • 扩大了变量的作用域，不利于提高程序的内聚性
  • 使用成员变量的几种情况
    • 描述某个类或某个对象的固有信息
    • 保存某个类或某个对象运行时的状态信息
    • 需要在某个类的多个方法之间进行共享
  • 局部变量的作用范围越小，它在内存里停留的时间就越短
    • 程序运行性能就越好
    • 能用代码块局部变量的地方，就坚决不要用方法局部变量

方法

[修饰符]  返回值类型  方法名（形参列表）
{
    //0 ~ n 个可执行语句
}
/*
形参列表：
0 ~ n 个：形参类型  形参名
*/
  • 修饰符
    • public 、protected 、private （最多只能出现其一）
    • final 、abstract （最多只能出现其一）
    • static （可以上面组合）
    • 或省略
  • 返回值类型
    • 基本类型
    • 引用类型
    • 方法体内必须有一个有效的 return 语句
    • 如果没有返回值
      • void 修饰
  • 方法名
    • 一个合法的标识符
    • 与成员变量名命名规则相同，但通常建议以英文动词开头（可读性）
  • 形参列表
    • 英文逗号 , 隔开
    • 谁调用方法，谁负责赋值形参
  • 方法不能独立存在
    • 只能在类体内定义
    • 逻辑上要么属于类，要么属于方法
    • 必须使用类或对象来调用
  • static 修饰的方法即可使用类调用，也可使用该的类的实例调用
    • 但使用该类的任何对象调用，结果一样
    • 因为底层依然是该类作为主调者
  • 非 static 修饰的方法只能使用实例（对象）来调用
    • 不同的对象调用，结果可能不同
  • 参数传递机制：值传递
    • 将实际参数值的副本（复制品）传入方法内
    • 参数本身不会受到任何影响

  • 形参个数可变的方法
    • 只能处于形参列表的最后
    • 一个方法中只能有一个个数可变的形参
/*
f(0)     = 1
f(1)     = 4
f(n + 2) = 2 * f(n + 1) + f(n)
递归调用形式：
f(n)     = 2 * f(n - 1) + f(n - 2)
*/
public class Dog
{
       public static int fn(int n)     //定义一个类方法 fn 返回 int 类型
       {
              if(n == 0)
              {
                  return 1;
              }

              else if(n == 1)
              {
                  return 4;
              }

              else
              {
                  return 2 * fn(n - 1) + fn(n - 2);
              }
              /*
              方法调用它自身
              向着已知的 f(0)=1 、 f(1)=4 方向递归
              */

       }

       public static void main(String[] args)
       {
              System.out.println(fn(10));         //输出：10497
       }
}



/*
f(20)     = 1
f(21)     = 4
f(n + 2) = 2 * f(n + 1) + f(n)
递归调用形式：
f(n)     = f(n + 2) - 2 * f(n + 1)
*/
public class Dog
{
       public static int fn(int n)     //定义一个类方法 fn 返回 int 类型
       {
              if(n == 20)
              {
                  return 1;
              }

              else if(n == 21)
              {
                  return 4;
              }

              else
              {
                  return fn(n + 2) - 2 * fn(n + 1);
              }
              /*
              方法调用它自身
              向着已知的 f(20)=1 、 f(21)=4 方向递归
              */

       }

       public static void main(String[] args)
       {
              System.out.println(fn(10));         //输出：-3771
       }
}
  • 递归

public class Dog
{
       public void test()                   //定义一个 test() 方法无参数
       {

               梦里不知身是客

}



//重载参数可变的形参
public class Dog
{
       public void test(String s1)          //定义第一个 test() 方法，有一个参数
       {
              System.out.println(s1);
       }
       public void test(String...s2)        //定义第二个 test() 方法，有参数个数可变的参数
       {
              System.out.println("$$$");
              for(String s : s2)
              {
                  System.out.println(s);
              }
       }

       public static void main(String[] args)
       {
              Dog d = new Dog();          //创建一个 Dog 类的实例对象并赋给 Dog 类型的变量 d

              d.test("独自莫凭栏");       //传入一个参数，将调用第一个 test() 方法

              d.test();                   //无参数传入，将调用第二个 test() 方法

              d.test("无限江山" , "别时容易见时难");   
               /*
               传入两个参数
               将调用第二个 test() 方法
               */

              d.test(new String[]{"流水落花春去也" , "天上人间"});
               /*
               传入数组
               将调用第二个 test() 方法
               */

               /*
               输出：
               独自莫凭栏
               $$$
               $$$
               无限江山
               别时容易见时难
               $$$
               流水落花春去也
               天上人间
               */
       }
}
  • 方法重载
    • 两同一不同
      • 同一个类
      • 同一个方法名
      • 不同的形参列表
      • 与方法的返回值、修饰符等没有任何关系
  • 一般不推荐重载形参个数可变的方法，容易降低程序的可读性

构造器

[修饰符]  构造器名（形参列表）
{
    //0 ~ n 个可执行语句
}
/*
形参列表：
0 ~ n 个：形参类型  形参名
*/
  • 修饰符
    • public 、protected 、private
    • 或省略
  • 构造器名
    • 必须和类名相同
  • 形参列表
    • 和定义方法的形参列表格式完全相同
  • 构造器不能定义返回值
    • 也不能使用 void 声明返回值
    • 如果定义了返回值或 void 声明，将被当作方法
  • 构造器是一个类创建对象的根本途径
    • 如果没有在类定义中编写构造器，系统会为该类提供一个默认的构造器
  • Java 通过 new 关键字来调用构造器
    • 返回该类的实例（对象）
public class HelloWorld
{
       public String s1;
       public String s2;
       public String s3;
       public String s4;
       public String s5;

       public HelloWorld(String...s)          //自定义一个构造器，包含一个参数可变的形参
       {
              this.s1  = s[0];
              this.s2  = s[1];
              this.s3  = s[2];
              this.s4  = s[3];
              this.s5  = s[4];              
       }

       public static void main(String[] args)                          
       {
              HelloWorld h = new HelloWorld("多少恨" , 
                                             "昨夜梦魂中" , 
                                             "还似旧时游上苑" , 
                                             "车如流水马如龙" , 
                                             "花月正春风");
              /*
              使用自定义的构造器来创建 HelloWorld 类的实例对象
              并传入自定义的初始值
              */

              System.out.println(h.s1);   
              System.out.println(h.s2); 
              System.out.println(h.s3); 
              System.out.println(h.s4); 
              System.out.println(h.s5); 
              /*
              输出：
              多少恨
              昨夜梦魂中
               还似旧时游上苑
               车如流水马如龙
               花月正春风
              */
       }
}
  • 一旦提供了自定义的构造器，系统就不再提供默认的构造器

  • 构造器重载
    • 两同一不同
      • 同一个类
      • 同一个类名
      • 不同的形参列表
  • 从而允许使用不同的构造器来初始化实例对象
    • 系统通过传入的实参列表来识别匹配
public class HelloWorld
{
       public String s1;
       public String s2;
       public String s3;
       public String s4;
       public String s5;

       public HelloWorld(){}

       public HelloWorld(String s1 , String s2)       
       {
              this.s1  = s1;

       }

       {

              通过 this 来调用第二个构造器里
              s1 、s2 的初始化代码
              */

              this.s3  = s3;
              this.s4  = s4;
              this.s5  = s5;              
       }

       public static void main(String[] args)                          
       {
              HelloWorld h1 = new HelloWorld();

              HelloWorld h2 = new HelloWorld("浪花有意千里雪" , 
                                              "桃花无言一队春");

              HelloWorld h3 = new HelloWorld("浪花有意千里雪" , 
                                              "桃花无言一队春" ,
                                             "一壶酒" , 
                                              "一竿身" , 
                                              "快活如侬有几人");

              System.out.println(h1.s1 + '\n' + 
                                  h1.s2 + '\n' + 
                                  h1.s3 + '\n' + 
                                  h1.s4 + '\n' + 
                                  h1.s5);

             null
             null
             null
             null
             null

             /*

             桃花无言一队春
             null
             null
             null
             */

  • 构造器 B 完全包含构造器 A
  • 构造器 B 内部调用构造器 A
    • 构造器不能直接调用，构造器必须使用关键字 new 来调用
      • 但这会导致系统创建一个实例对象
    • 为了在构造器 B 内调用构造器 A 的初始化代码，且不会创建实例对象
      • 可以使用关键字 this
      • 使用 this 调用另一个重载的构造器只能在构造器中使用
      • 且必须作为构造器执行体内的第一条语句
      • 系统会根据关键字 this 后面的括号内的实参，来匹配调用形参列表与之对应的构造器

软件开发的一个规则：不要把相同的代码段书写两次以上，以降低软件的维护成本

             */
    }

    • 类.类变量
    • 类.类方法
    • 实例.实例变量
    • 实例.实例方法
  • static 修饰的方法和成员变量
    • 可以通过类或实例调用
  • 没有 static 修饰的方法和成员变量
    • 只能通过实例调用

对象的this引用

  • this 关键字总是指向调用该方法的对象
    • 构造器中
      • 引用该构造器正在初始化的对象
    • 方法中
      • 引用调用该方法的对象
  • 最大的作用：让类中的一个方法，访问该类中的另一个方法或实例变量
//无 this
public class Dog
{
       public void jump()       //定义一个 jump() 方法
       {
              System.out.println("风回小院庭芜绿" + '\n' + "柳眼春相续");
       }

       public void run()        //定义一个 run()  方法，需要用到 jump() 方法
       {
              Dog d = new Dog();
              d.jump();
              /*
              jump() 方法没有使用 static 修饰
              必须用实例（对象）来调用
              因此创建了一个 Dog 类对象并赋给 Dog 类型的变量 d 
              用 d 来调用实例方法 jump()
              */

              System.out.println("凭阑半日独无言" + '\n' + "依旧竹声新月似当年");
       }


       public static void main(String[] args)
       {

              Dog dog = new Dog();   //创建了一个 Dog 类对象并赋给 Dog 类型的变量 dog              

              dog.run();
              /*
              输出：
              风回小院庭芜绿
              柳眼春相续
              凭阑半日独无言
              依旧竹声新月似当年
              dog 的 run() 方法需要依赖 d 的 jump() 方法
              不符合逻辑
              */
       }     
}



//用 this
public class Dog
{
       public void jump()       //定义一个 jump() 方法
       {
              System.out.println("风回小院庭芜绿" + '\n' + "柳眼春相续");
       }

       public void run()        //定义一个 run()  方法，需要用到 jump() 方法
       {
              this.jump();
              /*
              谁调用这个方法
              this 就代表谁
              且 this 只能代表当前类的实例
              this 可以省略
              */

              System.out.println("凭阑半日独无言" + '\n' + "依旧竹声新月似当年");
       }


       public static void main(String[] args)
       {

              Dog dog = new Dog();   //创建了一个 Dog 类对象并赋给 Dog 类型的变量 dog        

             dog.run();
             /*
             输出：
             风回小院庭芜绿
              柳眼春相续
              凭阑半日独无言
              依旧竹声新月似当年
              this 代表 dog
             */
       }     
}

  • 作为调用该方法的对象的引用
  • 调用 static 修饰的成员时省略了主调，默认使用该类作为主调
  • 调用非 static 修饰的成员时省略了主调，默认使用 this 作为主调
  • 小技巧
    • 看到对象调用 static 修饰的成员
    • 把其当作假象
    • 将其替换成用类来调用 static 修饰的成员
  • 如果确实需要在静态方法中访问另一个普通方法
    • 需要创建一个对象
    • new Dog().info()
      • 创建 Dog() 对象来调用 info() 方法
      • 输出：无言独上西楼，月如钩。寂寞梧桐深院，锁清秋。
  • 如果方法里有个局部变量和成员变量同名，程序又需要在该方法里访问这个被覆盖的成员变量
    • 则 this 不可省略
public class Dog
{
       public int foo;          //创建一个 foo 成员变量

       public Dog()             //构造器名必须与类名 Dog 相同，且不能有返回值类型
       {
              int foo  = 0;     //构造器内定义一个 foo 局部变量

              this.foo = 18;    
              /*
              this 代表该构造器正在初始化的对象
              且把对象的 foo 成员变量设置为18
              此处 this 不可省略
              因为有个成员变量名也为 foo
              */
       } 


       public static void main(String[] args)
       {
              System.out.println(new Dog().foo);
              /*
              构造器返回的所有对象的 foo 成员变量
              都等于18
              输出：18
              */
       }
}
  • 作为构造器初正在始化对象的引用
  • 与普通方法类似，大部分时候在构造器内访问其他成员变量和方法时都可以省略 this 前缀
    • 不可省略 this 的情况如上
public class Dog
{
       public int age;            //创建一个 age 成员变量

       public Dog grow()          //定义一个 grow 方法，返回值为 Dog 类型
       {
              age++;              //age 成员变量值加1

              return this;        //返回调用该方法的对象
       } 

       public static void main(String[] args)
       {
              Dog d = new Dog();

              d.grow()
               .grow()
               .grow()
               .grow();
              /*
              因为方法返回的是调用该方法的实例对象
              所以可以连续调用同一个方法
              */

              System.out.println(d.age);
              /*
              输出：4
              */
       }
}
  • 程序可以像访问普通变量一样访问 this 引用
  • 甚至可以把 this 当成普通方法的返回值
  • 返回的是调用该方法的实例对象

把该隐藏的隐藏，把该暴露的暴露

public class HelloWorld
{
       private String name;                                       //Private 修饰的实例变量 name

       private int age;                                           //Private 修饰的实例变量 age

       public void setName(String name)                           //public 修饰的 setter 
       {
              if(name.length() > 6 || name.length() < 2)
              {
                 System.out.println("您设置的人名不符合要求");
                 return;
              }
                else
              {
                 this.name = name;
                }
       }

       public String getName()                                    //public 修饰的 getter 
       {
              return this.name;
       }

       public void setAge(int age)                                //public 修饰的 setter 
       {
              if(age > 100 || age < 0)
              {
                  System.out.println("您设置的额年龄不符合要求");
                  return;
              }
              else
              {
                  this.age = age;
              }

       }

       public int getAge()                                        //public 修饰的 getter 
       {
              return this.age;
       }

       /*
       private 修饰的实例变量
       public 修饰的 setter 方法
       public 修饰的 getter 方法
       符合 JavaBean 规范的类
       JavaBean 总是一个封装良好的类
       */
}



public class Dog
{ 
       public static void main(String[] args)
       {
              HelloWorld h = new HelloWorld();

              //h.age = 1000;        //实例变量 age 已被封装隐藏，所以此句会出现编译错误

              h.setAge(1000);                       //输出：您设置的年龄不符合要求

              System.out.println(h.getAge());       //输出：0

       }

  • 成员访问控制符
    • private
      • 同一个类中
    • default
      • 同一个类中
      • 同一个包中的其他类
    • protected
      • 同一个类中
      • 同一个包中的其他类
      • 不同包中的子类
        • 修饰方法：希望其子类来重写这个方法
    • public
      • 同一个类中
      • 同一个包中的其他类
      • 不同包中的子类
      • 全局范围
  • 局部变量的作用域就是它所在的方法，不可能被其他类访问，因此不能使用访问控制符
  • 外部类
    • public
      • 可以被所有类使用
    • default
      • 同一个包中的其他类
    • 因为外部类没有处于任何类的内部
    • 也就没有所在类的内部、所在类的子类两个范围
  • 几个良好的建议：
    • 类里的绝大部分成员变量都应该使用 private 修饰
    • 只有一些 static 、类似全局变量的成员变量，才能考虑使用 public
    • 工具方法（只用于辅助实现该类的其他方法）也应该使用 private 修饰
    • 如果某个类只用做其他类的父类，该类的大部分方法可能希望被其子类重写，而不想被外界直接调用，则应该使用 protected 修饰
    • 希望暴露出来给其他类自由调用的方法应该用 public 修饰
    • 类的构造器应该使用 public 修饰，从而允许在其他地方创建该类的实例
      • 业务需要
        • protected
          • 主要用于被其子类调用
        • private
          • 阻止其他类创建该类的实例
    • 外部类应该使用 public 修饰，因为外部类通常都希望被其他类自由使用
