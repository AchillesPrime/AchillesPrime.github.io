继承

class Hello
{
      public String s1;

      public void info()
      {
             System.out.println(this.s1 + 
                                 "晚凉天净月华开" + '\n' + 
                                 "相得玉楼瑶殿影" + '\n' + 
                                 "空照秦淮");
      }
}


public class HelloWorld extends Hello                   //HelloWorld 子类继承了 Hello 父类
{
    public static void main(String[] args)
    {
           HelloWorld h = new HelloWorld();             //创建 HelloWorld 类的实例对象

           h.s1 = "金锁已沉埋" + '\n' + "壮气蒿莱";     //调用继承的 s1 成员变量

           h.info(); 
           /*
           调用继承的 info() 方法
           输出：
           金锁已沉埋
           壮气蒿莱
            晚凉天净月华开
            相得玉楼瑶殿影
           空照秦淮
           */
    }
}
  • Java 通过关键字 extends 实现继承
    • 被继承的类：父类
    • 实现继承的类：子类
    • 父类和子类的关系，是一种一般和特殊的关系
    • 父类的包含范围总是比子类包含的范围要大
      • 所以可以认为父类是大类，子类是小类
    • extends 关键字的英文翻译是“扩展”，而不是“继承”
      • 子类是对父类的扩展，子类是一种特殊的父类
      • 之所以翻译成“继承”，是因为子类可以获得父类所有的成员变量和方法
        • 子类不能获得父类的构造器
  • Java 不支持 C++ 那种多继承特征，仅支持单继承
    • 一个子类只有一个直接父类
    • 但一个子类可以有无限多个间接父类
    • 如果定义一个类时，未指定直接父类，则这个类默认扩展 java.lang.Object 类
    • java.lang.Object 类是所有类的父类（直接或间接）
      • 因此所有的类都可以调用 java.lang.Object 类所定义的实例方法
  • 子类扩展（extends）了父类，父类派生（derive）出了子类

class Hello
{
      public static void test(){...}         //类方法
}

class HelloWorld extends Hello
{
      public void test(){...}                //实例方法
}


//方法重写正确示例
class Hello
{
      private void test(){...}               //private 修饰的实例方法
}

class HelloWorld extends Hello
{
      public static void test(){...}         //新的类方法
}
  • 方法重写（方法覆盖）
    • 两同
      • 方法名相同
      • 形参列表相同
    • 两小
      • 子类方法返回值类型 <= 父类方法返回值类型
      • 子类方法声明抛出的异常类 <= 父类方法声明抛出的异常类
    • 一大
      • 子类方法的访问权限 >= 父类方法的访问权限
  • 覆盖方法和被覆盖方法要么都是类方法，要么都是实例方法
    • 不能一个是类方法，一个是实例方法
  • 父类中 private 修饰的方法对其子类是隐藏的，子类无法重写
    • 如果子类中定义了一个符合重写条件的方法，依然不是重写，而是定义了一个全新的方法
  • 方法重载和方法重写的区别
    • 重载 overload
      • 发生在同一个类中的多个同名方法之间
      • 子类获得了父类的方法，所以父类和子类方法之间也可能发生重载
    • 重写 override
      • 发生在子类和父类的同名方法之间

方法的super限定

class Hello
{
      public void fly()
      {
             System.out.println("辘轳金井梧桐晚" + 
                                "几树惊秋" + 
                               "昼雨新愁" +
                               "百尺虾须在玉钩");
      }
}


public class HelloWorld extends Hello                   //HelloWorld 子类继承了 Hello 父类
{

       public void fly()                                //父类方法 fly() 的重写
       {
              System.out.println("辘轳金井梧桐晚" + '\n' + 
                                 "几树惊秋" + '\n' + 
                                "昼雨新愁" + '\n' +
                                   "百尺虾须在玉钩");
       }

       public void flyfly() 
       {
              super.fly();                               //父类方法的调用
       }

       public static void main(String[] args)
       {
              HelloWorld h = new HelloWorld();           //创建 HelloWorld 类的实例对象

              h.fly(); 
              /*
              调用重写实例方法 fly() 
              输出：
              辘轳金井梧桐晚
               几树惊秋
               昼雨新愁
               百尺虾须在玉钩
              */

              h.flyfly();
              /*
              调用父类被覆盖的实例方法 fly()
              输出：
               辘轳金井梧桐晚几树惊秋昼雨新愁百尺虾须在玉钩
              */
    }
}
  • super 关键字用于限定该对象调用它从父类继承得到的被覆盖的实例方法
    • 类方法、类变量需通过父类类名调用
    • 同 this 一样， super 也不能出现在 static 修饰的方法中
      • static 修饰的方法属于类，该方法的调用者可能是一个类
        • 因而 super 限定也失去了意义
class Hello
{
      public int a = 18;
}


public class HelloWorld extends Hello                   //HelloWorld 子类继承了 Hello 父类
{
       public int a = 30;                         //子类实例变量 a 隐藏了父类实例变量 a

       public void fly()
       {
              System.out.println(a);
       }

       public void flyfly()
       {
              System.out.println(super.a);
       }


       public static void main(String[] args)
       {
              HelloWorld h = new HelloWorld();             //创建 HelloWorld 类的实例对象

              h.fly();
              /*
              访问子类中定义的实例变量 a
              输出：
              30
              */

              h.flyfly();
              /*
              访问父类中定义的被隐藏的实例变量 a
              输出：
              18
              */
    }
}

  • super 关键字用于限定该构造器初始化的是该对象从父类继承得到的被隐藏的实例变量，而不是该类自己定义的实例变量
    • 子类定义个和父类同名的实例变量
      • 子类实例变量会隐藏父类实例变量（不是覆盖）
  • 如果子类没有包含和父类同名的成员变量，则在子类中访问时无需使用 super 或父类名作为主调访问父类中定义的成员变量
  • a 成员变量的访问顺序
    • 查找该方法中是否有名为 a 的局部变量
    • 查找当前类中是否有名为 a 的成员变量
    • 查找直接父类中是否有名为 a 的成员变量，依次上溯所有的父类，直到 java.lang.Object 类
    • 编译错误
  • 当创建一个子类对象时，系统会为当前子类以及所有父类里定义的实例变量分配内存

  • 子类不会获得父类的构造器，但子类可以调用父类构造器的初始化代码
    • 在一个构造器中调用另一个重载的构造器，使用关键字 this
    • 在子类构造器中调用父类的构造器，使用关键字 super
      • 也必须出现在子类构造器执行体的第一行
      • 系统会根据关键字 super 后面的括号内的实参，来匹配调用父类中形参列表与之对应的构造器
class Hello
{
      public Hello()
      {
             System.out.println("Hello 类的无参数构造器");
      }
}

class HelloW extends Hello
{     
      public HelloW()
      {
             System.out.println("HelloW 类的无参数构造器");
      }

      public HelloW(String s)
      {
             System.out.println("HelloW 类的带一个参数 " + s + " 的构造器");
      }

      public HelloW(String s , String ss)
      {
             this(s);                     //this 关键字调用当前类中重载的构造器的初始化代码

             System.out.println("HelloW 类的带两个参数 " + s + "、" + ss + " 的构造器");
      }
}

public class HelloWorld extends HelloW
{       
       public HelloWorld()
       {
              //super("李煜" , "周邦彦");    //super 关键字调用父类中有两个参数的构造器

              System.out.println("HelloWorld 类的无参数构造器");
       }

       public static void main(String[] args)
       {
              new HelloWorld();
              /*
              输出：
              Hello 类的无参数构造器
              HelloW 类的带一个参数 李煜 的构造器
              HelloW 类的带两个参数 李煜 、周邦彦 的构造器
              HelloWorld 类的无参数构造器
              */
       }
}
  • 子类调用父类构造器的几种情况
    • 使用 super 关键字显示调用
      • 系统会根据关键字 super 后面的括号内的实参，来匹配调用父类中形参列表与之对应的构造器
    • 使用 this 关键字显示调用当前子类中重载的构造器
      • 系统会根据关键字 this 后面的括号内的实参，来匹配调用当前子类中形参列表与之对应的构造器
      • 执行另一个构造器时即会调用父类的无参数构造器
    • 既没有 super 调用也没有 this 调用
      • 系统会在执行子类构造器之前，隐式调用父类无参数构造器
  • 不管哪种情况
    • 父类构造器总会在子类构造器之前执行
    • 不停地上溯
    • 创建任何 Java 对象，最先执行的总是 java.lang.Object 类的构造器

多态

class Hello
{
      public int a = 10;

      public void base()
      {
             System.out.println("父类的实例方法");
      }

      public void test()
      {
             System.out.println("父类的被覆盖的实例方法");
      }
}

public class HelloWorld extends Hello
{       
       public String a = "周邦彦";

       public void test()
       {
              System.out.println("子类覆盖父类的实例方法");
       }
       public void sub()
       {
              System.out.println("子类的实例方法");
       }

       public static void main(String[] args)
       {
              Hello h1 = new Hello();

              System.out.println(h1.a);                   //输出：10

              h1.base();                                  //输出：父类的实例方法
              h1.test();                                  //输出：父类的被覆盖的实例方法

              HelloWorld h2 = new HelloWorld();   

              System.out.println(h2.a);                   //输出：周邦彦

              h2.base();                                  //输出：父类的实例方法
              h2.test();                                  //输出：子类覆盖父类的实例方法

              Hello h3 = new HelloWorld();

              System.out.println(h3.a);                   //输出：10

              h3.base();                                  //输出：父类的实例方法
              h3.test();                                  //输出：子类覆盖父类的实例方法
              //h3.sub();
              /*
              编译错误
              h3 编译时是 Hello 类型
              h3 运行时是 HelloWorld 类型
              没有定义 sub() 方法
              */
       }
}
  • Java 引用变量的两个类型
    • 编译时类型
      • 由声明该变量的类型决定
    • 运行时类型
      • 由实际赋给变量的类型决定
  • 编译时类型和运行时类型不一致，就会出现多态（Polymorphism）
    • Java 允许把一个子类的实例对象直接赋给一个父类的引用变量，且无需任何类型转换，或称向上转换（upcasting）
    • 总是由系统自动成功完成
      • 从侧面证实了子类是一种特殊的父类
    • 编译时是父类类型
    • 运行时是子类类型
    • 实际上是子类实例
    • 调用该引用变量的方法时，其行为总是表现出子类的行为特征
      • 这就可能出现：相同类型的变量、调用同一个方法时，呈现出多种不同的行为特征，这就是多态
    • 实例变量不具备多态性
      • 系统访问的是编译时类型的类中所定义的实例变量
  • Object p = new Person()
    • p 只能调用 Object 类中的方法
    • 不能调用 Person 类中的方法
      • 除非 Person 类中有 Object 类的方法重写
public class HelloWorld extends Hello
{       

       public static void main(String[] args)
       {
              Object obj = "Hello";

              String objStr =(String)obj;

              System.out.println(objStr);
              /*
              obj 编译时类型为 Object 类
              obj 运行时类型为 String 类
              Object 与 String 存在继承关系
              可以强制类型转换
              obj 实际类型为 String 类型
              输出：
              Hello
              */

              Object objPri = Integer.valueOf(5);
              /*
              obj 编译时类型为 Object 类
              obj 运行时类型为 Integer 类
              Object 与 Integer 存在继承关系
              可以强制类型转换
              objPri 实际类型为 Integer 类型
              */

              if(objPri instanceof String)
              {
                  String str =(String)objPri;
              }
              /*
              objPri 实际类型为 Integer 类型
              所以此选择条件为假
              即不可成功转换
              先用 instanceof 判断
              再用 type 转换
              从而保证程序不会出错
              无输出
              */
       }
}
  • 引用变量的强制类型转换
    • (type)variable
    • 只能在具有继承关系的两个类型之间转换
    • 试图把一个父类实例转换成子类类型，则这个对象必须实际是子类实例才行
      • 即编译时为父类类型
      • 运行时为子类类型
      • 否则将引发 ClassCastException 异常
public class HelloWorld extends Hello
{       

       public static void main(String[] args)
       {
              Object hello = "Hello";

              System.out.println("字符串是否是 Object 类的实例："     
                                 +  (hello instanceof Object));      //true

              System.out.println("字符串是否是 String 类的实例："     
                                 +  (hello instanceof String));      //true

              System.out.println("字符串是否是 Math 类的实例："       
                                 +  (hello instanceof Math));        //false

              System.out.println("字符串是否是 Comparable 类的实例：" 
                                 +  (hello instanceof Comparable));  //true

              String a = "Hello";

              System.out.println("字符串是否是 Math 类的实例："       
                                 +  (hello instanceof Math));        //false

              //System.out.println("字符串是否是 Math 类的实例："       
                                 +  (a instanceof Math)); 
               /*
               编译错误
               */

       }
}
  • instanceof
    • 前一个操作数
      • 引用类型变量
    • 后一个操作数
      • 类、接口（可以把接口理解成一个特殊的类）
    • 用于判断前面的对象是否是后面类的实例，或者其子类、实现类的实例
      • 是否可以成功转换，从而保证程序更加健壮
    • 如果是，返回 true
    • 如果不是，返回 false
  • 前面操作数的编译时类型
    • 要么与后面的类相同
    • 要么与后面的类具有父子继承关系
    • 否则会引起编译错误

继承与组合

class Hello
{
      public Hello()
      {
             test();
      }

      public void test()
      {
             System.out.println("将被子类重写的方法");
      }
}

public class HelloWorld extends Hello
{       
       private String name;

       public void test()
       {
              System.out.println("子类重写父类的方法:" + name.length());
       }

       public static void main(String[] args)
       {
              //HelloWorld h = new HelloWorld();
              /*
              创建 HelloWorld 类的实例对象时
              会先执行其父类 Hello 的构造器
              父类 Hello 的构造器内调用了被其子类重写的实例方法 test()
              则变成调用子类重写的实例方法 test()
              而此时 HelloWorld 类的实例对象 h 的实例变量 name 值为 null
              所以会引发空指针异常：
              NullPointerException
              */          
       }
}
  • 继承是实现类复用的重要手段，但带来了一个最大的坏处
    • 破坏封装
    • 造成子类和父类的严重耦合，父类的实现细节对子类不再透明，子类可以恶意篡改父类的方法
  • 设计父类通常应该遵循如下规则
    • 尽量隐藏父类的内部数据
    • 尽量把所有成员变量都设置为 private 访问类型，不让子类直接访问父类的成员变量
    • 不让子类随意访问、修改父类的方法
    • 父类中辅助其他方法的工具方法，应该都使用 private 访问控制符修饰
    • 父类中供外部类调用的方法设置为 public 的同时，用 final 修饰，让子类无法重写
    • 如果希望父类中的方法被重写，但又不希望被其他类自由访问，应该用 protected 访问控制符修饰
    • 尽量不要在父类构造器中调用将要被子类重写的方法
  • 如果想把某个类设置成最终类（即不能被当作父类）
    • 可以使用 final 修饰
    • 例如 JDK 提供的 java.lang.String 和 java.lang.System
    • 或者可以使用 private 修饰这个类的所有构造器，从而保证子类无法调用该类的构造器，也就无法继承这个类
      • 对于把所有构造器都用 private 修饰的类，可以另外提供一个静态方法，用于创建该类的实例
    • 到底何时需要使用父类派生子类
      • 子类需要额外增加属性，而不仅仅时属性值的改变
      • 子类需要额外增加自己独有的行为特征（包括新增的方法或重写父类的方法）
//通过继承复用
class Hello
{
      private void A()
      {
              System.out.println("别来春半" + '\n' + "触目柔肠断");
      }

      public void B()
      {
             A();

             System.out.println("砌下落梅如雪乱" + '\n' + "拂了一身还满");
      }
}

class HelloW extends Hello
{
      public void C()
      {
             System.out.println("$$$");
      }
}

public class HelloWorld extends Hello
{       
       public void D()
       {
              System.out.println("￥￥￥");
       }

       public static void main(String[] args)
       {
              HelloW h1 = new HelloW();
              h1.B();
              h1.C();
              /*
              通过继承 Hello 类
              使用实例方法 B()
              输出：
              别来春半
               触目柔肠断
               砌下落梅如雪乱
               拂了一身还满
               $$$
              */

              HelloWorld h2 = new HelloWorld();
              h2.B();
              h2.D();
              /*
              通过继承 Hello 类
              使用实例方法 B()
              输出：
              别来春半
               触目柔肠断
               砌下落梅如雪乱
               拂了一身还满
               ￥￥￥
              */
       }
}



//通过组合复用
class Hello
{
      private void A()
      {
              System.out.println("雁来音信无凭");
      }

      public void B()
      {
             A();

             System.out.println("路遥归梦难成" + '\n' +
                                "离恨恰如春草" + '\n' +
                                 "更行更远还生");
      }
}

class HelloW
{
      private Hello L;          //将原来的父类组合到原来的子类中，作为子类的一个组合部分

      public HelloW(Hello L)
      {
             this.L = L;
      }

      public void B()           //重新定义一个自己的 B() 方法
      {
             L.B();            //直接复用原来的父类的 B() 方法来实现子类 HelloW 的 B() 方法
      }

      public void C()
      {
             System.out.println("$$$$$");
      }
}

public class HelloWorld
{       
       private Hello L;         //将原来的父类组合到原来的子类中，作为子类的一个组合部分

       public HelloWorld(Hello L)
       {
              this.L = L;
       }

       public void B()          //重新定义一个自己的 B() 方法
       {
               L.B();            //直接复用原来的父类的 B() 方法来实现子类 HelloWorld 的 B() 方法
       }

       public void D()
       {
              System.out.println("￥￥￥￥￥");
       }

       public static void main(String[] args)
       {
              Hello h1 = new Hello();               //需要显式创建被组合的父类的对象

              HelloW h2 = new HelloW(h1);           //通过父类的对象来创建子类的对象
              h2.B();
              h2.C();
              /*
              输出：
              雁来音信无凭
               路遥归梦难成
               离恨恰如春草
               更行更远还生
              $$$$$
              */

              Hello h3 = new Hello();               //需要显式创建被组合的父类的对象

              HelloWorld h4 = new HelloWorld(h3);   //通过父类的对象来创建子类的对象
              h4.B();
              h4.D();
              /*
              输出：
              雁来音信无凭
               路遥归梦难成
               离恨恰如春草
               更行更远还生
              ￥￥￥￥￥
              */
       }
}
  • 组合也是实现类复用的重要方式，并能提供更好的封装性
  • 把该类当成另一个类的组合成分，从而允许新的类直接复用该类的 public 方法
  • 用户看到的是新类的方法，而不能看到被组合对象的方法
    • 因此通常需要在新类里使用 private 修饰被组合的旧类对象
  • is a
    • “是”
    • 对已有的类做一番改造，以此获得一个特殊的版本
    • 将一个较为抽象的类改造成能适用于某些特定需求的类
  • has a
    • “有”
    • 如果两个类之间有明显的整体、部分的关系
    • 可以把部分作为整体的组合成员变量，借助于部分的方法来实现整体的方法

初始化块

[修饰符]  
{
       0 ~ n 条初始化块的可执行性代码
}
  • Java 类里的第四种成员（前三种是成员变量、方法、构造器）
  • 前面定义的初始化块先执行，后面定义的初始化块后执行
  • 修饰符
    • 省略：普通初始化块
      • 负责对对象执行初始化
    • static：静态初始化块
      • 类初始化块
      • 负责对类进行初始化
  • 类初始化块在类初始化阶段执行，普通初始化块在创建对象时执行
    • 类初始化块先于普通初始化块执行
  • 类初始化块通常用于对类变量执行初始化处理，不能对实例变量进行初始化处理
    • 遵循静态成员不能访问非静态成员的规则
public class HelloWorld
{       
       {
             int a = 6;

             if(a > 4)
             {
                 System.out.println("初始化块的局部变量 a 大于 4");
             }

             System.out.println("第一个初始化块");
       }

       {
             System.out.println("第二个初始化块");
       }

       public HelloWorld()
       {
              System.out.println("无参数构造器");
       }

       public static void main(String[] args)
       {
              new HelloWorld(); 
              /*
              输出：
              初始化块的局部变量 a 大于 4
              第一个初始化块
              第二个初始化块
              无参数构造器
              */
       }
}
  • 当创建 Java 对象的时候，总是先调用该类初始化块
    • 无法通过类、对象来调用初始化块，只在创建对象时隐式执行，并先于构造器
    • 初始化块不能接受任何参数，对同一个类的所有对象所进行的初始化处理完全相同
      • 如果多个构造器中有相同的初始化代码，且无须接受参数，可提取成一个初始化块
      • 更好地提高初始化块代码的复用，提高整个应用的可维护性
  • 可以把多个普通初始化块合并成一个初始化块，从而让程序更加简洁、可读性更强
  • 与构造器类似，普通初始化块会一直上溯到 java.lang.Object 类的初始化块，然后一直执行下来
class Hello
{
      static
      {
             System.out.println("Hello 类的静态初始化块");
      }

      {
             System.out.println("Hello 类的普通初始化块");
      }

      public Hello()
      {
             System.out.println("Hello 类的无参数构造器");
      }
}

class HelloW extends Hello
{
      static
      {
             System.out.println("HelloW 类的静态初始化块");
      }

      {
             System.out.println("HelloW 类的普通初始化块");
      }

      public HelloW()
      {
             System.out.println("HelloW 类的无参数构造器");
      }

      public HelloW(String s)
      {
             this();         //通过 this 调用同一类中重载的构造器

             System.out.println("HelloW 类的带参数构造器,参数值为：" + s);
      }
}

public class HelloWorld extends HelloW
{       
       static
      {
             System.out.println("HelloWorld 类的静态初始化块");
      }

      {
             System.out.println("HelloWorld 类的普通初始化块");
      }

      public HelloWorld()
      {
             super("007");      //通过 super 调用父类中只有一个字符串参数的构造器

             System.out.println("HelloWorld 类的无参数构造器");
      }

      public static void main(String[] args)
      {
             new HelloWorld();

             new HelloWorld();

             /*
             类初始化阶段   （静态初始化块）
             对象初始化阶段 （普通初始化块、构造器）
             输出：

             Hello       类的静态初始化块
             HelloW      类的静态初始化块
             HelloWorld  类的静态初始化块

             Hello 类的普通初始化块
             Hello 类的无参数构造器
             HelloW 类的普通初始化块
             HelloW 类的无参数构造器
             HelloW 类的带参数构造器,参数值为：007             
             HelloWorld 类的普通初始化块
             HelloWorld 类的无参数构造器

             Hello 类的普通初始化块
             Hello 类的无参数构造器             
             HelloW 类的普通初始化块
             HelloW 类的无参数构造器
             HelloW 类的带参数构造器,参数值为：007             
             HelloWorld 类的普通初始化块
             HelloWorld 类的无参数构造器
             */
      }
}
  • 当 JVM 第一个主动使用某个类时，系统会在类准备阶段为该类的所有类变量分配内存，在初始化阶段则负责初始化这些类变量
  • 类的静态初始化块也会一直上溯到 java.lang.Object 类的静态初始化块，全部的类初始化块执行完后，才可以在系统中使用类
  • 类初始化成功后，将一直存在于虚拟机中，因此第二次创建类实例对象时，无需再次对类进行初始化
public class HelloWorld
{       

      {
             a = 6;        //普通初始化块将 a 实例变量赋值为 6
      }

      int a = 9;

      static
      {
             b = 6;        //静态初始化块将 a 类变量赋值为 6
      }

      static int b = 9;

      public static void main(String[] args)
      {
             System.out.println(new HelloWorld().a);

            System.out.println(HelloWorld.b);

             /*
             输出：
             9
             9
             */
      }
}
  • 普通初始化块与声明实例变量时指定初始值的执行顺序，与它们在源代码中的排列顺序相同
  • 类初始化块与声明类变量时指定初始值的执行顺序，与它们在源代码中的排列顺序相同
