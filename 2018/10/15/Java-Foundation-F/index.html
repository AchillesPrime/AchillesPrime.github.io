抽象类

abstract class H                              //抽象类
{
         {
            System.out.println("执行抽象类 H 的初始化块");
         }                          //初始化块，创建其子类实例对象时调用

         private String color;

         public abstract double C();         //抽象方法

         public abstract String getType();   //抽象方法

         public H(){}            //构造器，创建其子类实例对象时调用

         public H(String color)  //构造器，创建其子类实例对象时调用
         {
                System.out.println("执行抽象类 H 的构造器");
                this.color = color;
         }

         public void setColor(String color)
         {
                this.color = color;
         }

         public String getColor()
         {
                return this.color;
         }
}

class Triangle extends H     
/*
普通类
继承了抽象类 H
必须实现抽象类 H 中的所有抽象方法
*/
{
      private double a;
      private double b;
      private double c;

      public Triangle(String color , double a , double b , double c)
      {
             super(color);
             this.setSides(a , b , c);
      }

      public void setSides(double a , double b , double c)
      {
             if(a >= b + c || b >= a + c || c >= a + b)
             {
                 System.out.println("三角形的两边之和必须大于第三边");
                 return;
             }

             this.a = a;
             this.b = b;
             this.c = c;
      }

      public double C()
      {
             return a + b + c;
      }

      public String getType()
      {
             return "三角形";
      }

}

  • 抽象类和抽象方法（abstract 修饰）
    • 抽象类
      • 可以没有抽象方法
      • 不能被实例化
      • 不能使用 new 关键字调用抽象类的构造器创建抽象类的实例对象
        • 构造器主要是被其子类调用
      • 可以包含成员变量、普通方法和抽象方法、构造器、初始化块、内部类（接口、枚举）
    • 抽象方法
      • 直接定义一个抽象方法
      • 继承了一个抽象父类，但没有完全实现（重写）抽象父类包含的全部抽象方法
      • 实现了一个接口，但没有完全实现（重写）接口包含的全部抽象方法
      • 只能被定义在抽象类内
      • 不能有方法体
        • 抽象方法： public abstract void test();
        • 空方法体： public void test(){}
  • 关于修饰符的注意要点
    • abstract 修饰的类只能被继承，final 修饰的类不能被继承
    • abstract 修饰的方法必须由子类提供重写，final 修饰的方法不能被重写
      • abstract 和 final 不能共存
    • abstrcat 不能修饰成员变量、局部变量、构造器
    • static 修饰的方法属于类本身，通过类调用
    • abstract 修饰的方法为抽象方法，没有方法体
      • 通过类调用一个没有方法体的方法会出现编译错误
      • static 和 absract 不能共同修饰一个方法
      • 但 static 和 abstract 能共同修饰一个内部类
    • abstract 修饰的方法必须由子类重写才有意义
      • abstract 和 private 访问权限不能共存
abstract class H
{
         private double turnRate;

         public H(){}

         public abstract double C();       //抽象方法

         public void setTurnRate(double turnRate)
         {
                this.turnRate = turnRate;
         }

         public double getSpeed()
         {
                return turnRate * C();     
                /*
                速度等于（转速 * 周长）
                普通方法依赖抽象方法
                抽象方法推迟到子类中实现
                */
         }
}

public class HelloWorld extends H 
{
       private double r;

       public HelloWorld(double r)
       {
              this.r = r;
       }

       public double C()              //实现父类的抽象方法
       {
              return 2 * Math.PI * r;
       }

       public static void main(String[] args)
       {
              HelloWorld h = new HelloWorld(3);

              h.setTurnRate(15);

              System.out.println(h.C());
              System.out.println(h.getSpeed());
              /*
              输出：
              18.84955592153876
              282.7433388230814
              */
       }
}
  • 抽象类是从多个具体类中抽象出来的父类，它具有更高层次的抽象
  • 以抽象类作为子类的模板，可避免子类设计的随意性
  • 抽象类体现的就是一种模板模式的设计
    • 抽象类作为子类的通用模板，子类在抽象类的基础上进行扩展、改造
    • 抽象父类提供了子类的通用方法，并把一个或多个不能实现的方法抽象成抽象方法，实现的具体细节留给子类实现
  • 利用抽象类和抽象方法，可以更好地发挥多态的优势，使程序更加灵活

定义接口

  • interface
    • 接口定义了一种规范
    • 类是一种具体实现体
    • 接口不关心这些类里面的状态数据，也不关心这些类里面的实现细节
    • 接口只规定这些类里面必须提供某些方法
  • 接口是从多个相似类中抽象出来的公共行为规范，接口不提供任何实现，接口体现的是规范和实现分离的设计哲学
  • 充分利用接口可以极好地降低程序各模块之间的耦合，从而提高系统的可扩展性和可维护性
[修饰符]  interface  接口名  extends  父接口1、父接口2...
{
        0 ~ n 个静态常量
        0 ~ n 个抽象方法
        0 ~ n 个内部类、接口、枚举...
        0 ~ n 个私有方法、默认方法、类方法...
}

  • 修饰符

    • public
    • 或省略

  • 接口名

    • 与类名采用相同的命名规则
    • 接口名通常可以使用形容词
    • 如果一个 Java 源文件里定义了一个 public 接口，则此源文件的主文件名必须与该接口名相同

  • 一个接口可以有多个直接父接口，但接口只能继承接口，不能继承类

  • 接口里的成员都是 public 访问权限，定义时可以省略修饰符，显式指定修饰符则只能使用 public

    • 接口里的静态常量是与接口相关的
      • 系统会自动为这些成员变量增加 public static final 修饰符
        • 因此即使另一个类处于和接口不同的包下，也可以通过接口来访问接口里的成员变量：包名.接口名.静态常量名
      • 接口里面不包含构造器和初始化块
      • 成员变量只能在声明定义处指定初始值
    • 接口里的方法只能是抽象方法、私有方法、默认方法、类方法
      • 如果定义的方法不是私有方法、默认方法、类方法
      • 系统会自动为普通方法增加 public abstract 修饰符
        • 即接口里的普通方法不能有方法体，因为会自动升级到抽象方法
    • 接口里的内部类、内部接口、内部枚举
      • 系统会自动为之增加 public static 修饰符
    • 私有方法、默认方法、类方法必须有方法体
      • 私有方法
        • 必须使用 private 显式修饰
        • 主要作用就是作为被隐藏的工具方法，为接口中的默认方法和类方法提供支持
        • 不能使用 default 修饰
        • 可以使用 static 修饰
          • 私有方法既可以是类方法，也可以是实例方法
      • 默认方法
        • 必须使用 default 显式修饰
        • 系统会自动为默认方法增加 public 修饰符
        • 不能使用 static 修饰
          • 因此不能直接使用接口来调用默认方法
          • 需要使用接口的实现类的实例对象来调用默认方法
          • 接口的默认方法其实就是实例方法
            • 因为接口中的实例方法不能有方法体，所以只好重新定义一种所谓的“默认方法”，默认方法就是有方法体的实例方法
      • 类方法
        • 必须使用 static 显式修饰
        • 不能使用 default 修饰
        • 系统会自动为默认方法增加 public 修饰符
        • 类方法可以直接使用接口调用
interface A
{
          int a = 5;
          void testA();
}

interface B
{
          int b = 6;
          void testB();
}

interface C extends A , B     //接口 C 继承了接口 A 、接口 B
{
          int c = 7;
          void testC();
}

public class HelloWorld
{
       public static void main(String[] args)
       {
              System.out.println(C.a);
              System.out.println(C.b);
              System.out.println(C.c);
              /*
              通过接口 C 访问静态常量 a 、b 、c
              输出：
              5
              6
              7
              */
       }
}
  • 接口完全支持多继承
    • 一个接口可以有多个直接父接口
    • 多个父接口之间用英文逗号 , 隔开
    • 子接口扩展某个父接口，将会获得父接口的所有静态常量、抽象方法

使用接口

[修饰符]  class  类名  extends 父类  implements  接口1 , 接口2...
{
         类体部分
}
  • 接口的主要用途
    • 被实现类实现
    • 调用接口中定义的静态常量
    • 接口不能用于创建实例对象，但可以声明引用类型变量，也可以进行强制类型转换
      • 这个引用类型变量必须引用到该接口的实现类的实例对象
  • 一个类可以实现一个或多个接口（为单继承灵活性不足所做的补充）
    • 继承使用 extends
    • 实现使用 implements
  • 实现接口和继承父类相似，一样可以获得接口里的静态常量、抽象方法和实例方法
    • 一个类实现了一个或多个接口之后，这个类必须完全实现（重写）这些接口里所定义的全部抽象方法
    • 否则，该类将保留从父接口那里继承的抽象方法，该类也必须定义成抽象类
    • 可以把实现接口理解为一种特殊的继承，相当于实现类继承了一个彻底抽象的类（相当于除默认方法外，所有方法都是抽象方法的类）
interface A
{
        int MAX_LINE = 50;     //定义 public static final 静态常量

        void out();            //定义 public abstract 抽象方法

        void getDate(String s);        //定义 public abstract 抽象方法

        default void print(String...s)     //定义 public default 默认方法
        {
                for(String S : s)
                {
                    System.out.println("打印：" + '\t' + S);
                }
        }

        default void defaultTest()         //定义 public default 默认方法
        {
                System.out.println("默认方法 defaultTest()");
        }

        static String staticTest()         //定义 public static 类方法
        {
               return "接口里的类方法";
        }

        private void foo()                 //定义 public private 私有方法
        {
                System.out.println("私有方法 foo()");
        }

        private static void bar()          //定义 public private static 私有类方法
        {
                System.out.println("私有类方法 bar()");
        }
}

interface B
{
        int getTime();          //定义 public abstract 抽象方法
}

public class HelloWorld implements A , B
{
       private String[] printData = new String[MAX_LINE];   //定义打印作业的数组列队       

       private int dataNum = 0;     //记录需要打印的作业数

       public void out()
       {
              while(dataNum > 0)    //只要还有作业，就继续打印
              {
                  System.out.println("打印：" + '\t' + printData[0]);
                  System.arraycopy(printData , 1 , printData , 0 , --dataNum);
                  /*
                  把作业的数组列队整体向前移一位
                  并将剩下的作业数减1
                  */
              }
       }

       public void getDate(String s)
       {
              if(dataNum >= MAX_LINE)
              {
                  System.out.println("输出队列已满，添加失败");
              }
              else
              {
                  printData[dataNum++] = s;      
                  /*
                  把打印作业添加到数组列队里
                  需要打印的作业数加1
                  */
              }
       }

       public int getTime()
       {
              return 45;
       }

       public static void main(String[] args)
       {
              A a = new HelloWorld();
              /*
              创建一个 HelloWorld 实现类的实例对象
              当作接口 A 用
              */

              a.getDate("花明月暗笼轻雾");
              a.getDate("今宵好向郎边去");
              a.out();

              a.getDate("刬袜步香阶");
              a.getDate("手提金缕鞋");
              a.out();

              a.print("画堂南畔见" , "一向偎人颤" , "奴为出来难" , "教君恣意怜");

              a.defaultTest();
              /*
              直接调用 print() 和 defaultTest() 默认方法
              */

              B b = new HelloWorld();

              System.out.println(b.getTime());

              Object o = b;

              /*
              总输出：
              打印：  花明月暗笼轻雾
              打印：  今宵好向郎边去
              打印：  刬袜步香阶
              打印：  手提金缕鞋
              打印：  画堂南畔见
              打印：  一向偎人颤
              打印：  奴为出来难
              打印：  教君恣意怜
              默认方法 defaultTest()
              45
              */
       }
}
  • 实现接口方法时，必须使用 public 访问控制修饰符
    • 接口里的方法都是 public
    • 实现类重写接口里的方法规定：
      • 子类方法的访问权限 >= 父类方法的访问权限
  • 接口不能显式继承任何类
    • 但所有的接口类型的引用变量都可以直接赋值给 Object 类型的引用变量
    • 向上转型的自动实现
      • 因为编译器知道，任何 Java 实例对象都必须是 Object 或其子类的实例对象，任何接口的实现类的实例对象也不例外，该实现类肯定是 Object 类的显式或隐式子类

接口和抽象类

  • 相似点：
    • 都不能被实例化
    • 都位于继承树的顶端，用于被其他类实现和继承
    • 都可以包含抽象方法，实现接口或继承抽象类的普通子类都必须实现这些抽象方法
  • 主要差别（体现在二者的设计目的上）
    • 接口作为系统与外界交互的窗口，体现的是一种规范
      • 接口规定了实现者必须向外提供哪些服务
      • 接口规定了调用者可以调用那些服务和如何调用这些服务
      • 在一个程序中使用接口，接口是多个模块之间的耦合标准
      • 在多个应用程序中使用接口，接口是多个应用程序的通信标准
      • 接口类似于整个系统的“总纲”，它制订了系统各模块应该遵循的标准，因此一个系统中的接口不应该经常改变
      • 一旦接口被改变，造成的影响是辐射式的，导致系统中大部分的类都需要改写
    • 抽象类作为系统中多个子类的共同父类，体现的是一种模板式设计
      • 可以被当作系统实现过程中的中间产品
      • 实现了系统的部分功能，但不能当成最终产品，必须有更进一步的完善
  • 其他差别
    • 接口里只能包含抽象方法、私有方法、默认方法和类方法，不能为普通方法提供方法实现
    • 抽象类则可以完全包含普通方法，甚至一个普通类（没有包含抽象方法的类）增加一个 abstract 修饰符后就将直接变成抽象类
    • 接口里只能定义静态常量，不能定义普通成员变量
    • 抽象类里既可以定义静态常量，也可以定义普通成员变量
    • 接口里不包含构造器
    • 抽象类里可以包含构造器，让其子类调用这些构造器来完成属于抽象类的初始化操作
    • 接口里不包含初始化块
    • 抽象类里可以包含初始化块
    • 一个类只能有一个直接父类，包括抽象类
    • 一个类可以实现多个接口，通过实现多个接口可以弥补 Java 单继承的不足

内部类

  • 把一个类放在另一个类的内部定义
  • 定义在其他类内部的类就被称为内部类（或嵌套类）
  • 包含内部类的类被称为外部类（或宿主类）
  • 内部类提供了更好的封装，不允许同一个包中的其他类访问
  • 内部类被当成其外部类的成员，可以直接访问外部类的私有数据
    • 同一个类的成员之间可以互相访问
    • 但外部类不能访问内部类的实现细节，例如内部类的成员变量
  • 匿名内部类适合用于创建那些仅需要使用一次的类
  • 内部类比外部类可以多使用三个修饰符
    • private
    • protected
    • static
    • 外部类不可使用这三个修饰符
    • 外部类的上一级程序单元是包，所以只有两个作用域，对应两种访问控制符
      • 同一个包（default ，即省略）
      • 任何位置（public）
    • 内部类的上一级程序单元是外部类，所以有四个作用域，对应四种访问控制符
      • 同一个类（private）
      • 同一个包（default ，即省略）
      • 父子类（protected）
      • 任何位置（public）
  • 成员内部类是一种与成员变量、方法、构造器和初始化块相似的类成员
    • 包括非静态内部类和静态内部类
    • 局部内部类和匿名内部类不是类成员，即不是成员内部类
  • 外部类的子类里不可重写父类的内部类
    • 内部类的类名不再是简简单单的类名
      • 它实际上还把外部类的类名作为一个命名空间，作为内部类类名的限制
    • 子类中的内部类和父类中的内部类即使同名
      • 但它们所处的外部类空间不同，所以它们不可能完全同名，也就不可能重写
public class HelloWorld
{       
       private double weight;

       public HelloWorld(){}

       public HelloWorld(double weight)
       {
              this.weight = weight;
       }

       private class Hello    //定义非静态内部类
       {
               private double length;     //省略了 setter 和 getter 方法
               private String color;      //省略了 setter 和 getter 方法

               public Hello(){}

               public Hello(double length , String color)
               {
                      this.length = length;                      
                      this.color  = color;
               }

               public void info()
               {
                      System.out.println("颜色：" + color + 
                                          '\n' +  
                                           "高：" + length);

                      System.out.println("重：" + weight);
                        /*
                        直接访问外部类的私有实例变量 weight
                        因为在非静态内部类 Hello 的实例对象里
                        保存了一个它所寄生的外部类 HelloWolrd 实例对象的引用
                        当调用非静态内部类 Hello 的实例方法时
                        必须有一个非静态内部类 Hello 的实例对象
                        非静态内部类 Hello 的实例对象必须寄生在
                        外部类 HelloWorld 的实例对象里
                        */
               }
       }

       public void test()
       {
              Hello o = new Hello(10 , "黑白相间");
              o.info();
       }

       public static void main(String[] args) 
       {              
              HelloWorld h = new HelloWorld(500);
              h.test();
              /*
              输出：
              颜色：黑白相间
              高：10.0
              重：500.0
              */
       }
}

一个是 HelloWolrd$Hello.class
为内部类 Hello 的 *.class 字节码文件
成员内部类的 *.class 字节码文件总是遵循如下的命名格式：

       private class Hello

               public void info()

                        private int a = 999;

                        非静态内部类的私有成实例变量 a 
                        且非静态内部类里不能有静态成员
                      */

                      System.out.println(s);

                      System.out.println(this.s);

                      System.out.println(HelloWorld.this.s);
               }
       }

       public void test()
       {
              Hello o = new Hello();
              o.info();

               System.out.println(new Hello().a);

              在外部类 HelloWorld 中显式创建非静态内部类 Hello 的实例对象
              才能访问非静态内部类 Hello 的实例变量 a 
              且此行代码在静态方法 main 中将编译错误
              根据静态成员不能访问非静态成员的规则
              外部类的静态方法、静态代码块都不能访问非静态内部类
              包括不能使用非静态内部类定义的成员变量、创建非静态内部类实例等
              总之
              不允许在外部类的静态成员中直接使用非静态内部类
              所以此 test() 方法为非静态方法
              */
       }

       public static void main(String[] args) 
       {

              方法里的局部变量
              内部类的实例变量

              */

/*
系统会优先查找方法里的局部变量
如果没有找到，查找内部类的成员变量
如果没有找到，查找外部类的成员变量
如果依然没有找到
系统将出现编译错误：提示找不到该变量
通过 this.成员变量名 
访问非静态内部类的同名成员变量
通过 OuterClass.this.成员变量名 
访问外部类的同名成员变量
*/
  • 非静态内部类
    • 没有使用 static 修饰的成员内部类
  • 静态内部类（类内部类）
    • 使用 static 修饰的成员内部类
    • 属于外部类本身，而不属于外部类的某个实例对象
  • static 关键字的作用是把类的成员变成类相关，而不是实例相关
    • 外部类的上一级程序单元是包，所以不可使用 static
    • 内部类的上一级程序单元是外部类，使用 static 可以将内部类变成外部类相关，而不是外部类实例相关
  • 静态内部类可以包含静态成员，也可以包含非静态成员
  • 静态内部类不可访问外部类的实例成员，只能访问外部类的类成员
  • 静态内部类是外部类的一个静态成员，因此外部类的所有方法、所有初始化块中都可以使用静态内部类来定义变量、创建实例对象等
  • 外部类依然不能直接访问静态内部类的成员，但可以使用静态内部类的类名作为调用者来访问静态内部类的成员，也可使用静态内部类的对象作为调用者来访问静态内部类的实例成员
  • Java 允许在接口中定义内部类，默认使用 public static 修饰
    • 接口里内部类只能是静态内部类
    • 甚至接口里还能定义内部接口，默认使用 public static 修饰
      • 但定义内部接口的意义不大，毕竟接口的作用是定义一个公共规范，暴露出来供大家使用

/*
从上面的代码易看出
当定义一个静态内部类时
其外部类非常像一个包空间
相比之下
使用静态内部类比使用非静态内部类要简单很多
只要吧外部类当作一个包空间即可
因此当程序需要使用内部类时
应该优先考虑使用静态内部类
*/
  • 在外部类以外使用非静态内部类和静态内部类
    • 则不能使用 private 修饰
      • private 修饰的内部类只能在外部类内使用
    • default
      • 只能被在与外部类同一个包中的其他类访问
    • protected
      • 可被与外部类同一个包中的其他类和外部类的子类访问
    • public
      • 可以在任何地方被访问
  • 在外部类以外的地方定义静态内部类和非静态内部类的变量的方法：
    • OuterClass.InnerClass varName
      • 如果外部类有包名，应该增加包名前缀
  • 在外部类以外创建非静态内部类的实例对象的方法：
    • OuterInstance.new InnerConstructor()
    • 非静态内部类的实例对象必须寄生在外部类的实例对象里
      • 因此创建非静态内部类实例对象之前，必须先创建其外部类的实例对象
    • 非静态内部类的构造器必须使用外部类实例对象来调用
  • 在外部类以外创建静态内部类的实例对象的方法：
    • new OuterClass.InnerConstructor()
    • 静态内部类的实例对象不寄生在外部类的实例对象里
      • 因此创建静态内部类实例对象之前，无须先创建其外部类的实例对象
  • 综上
    • 声明变量
      • 非静态内部类和静态内部类完全一样
    • 调用构造器创建实例对象
      • 非静态内部类通过外部类的实例对象
      • 静态内部类通过外部类

class subIn extends Out.In         //非静态内部类 In 的子类
{
      public subIn(Out out)        //传入外部类 Out 的实例对象 out 
      {
          out.super("hello~");     
          /*
          super 关键字代表调用非静态内部类 In 的构造器
          非静态内部类 In 的构造器必须使用外部类 Out 的实例对象 out 来调用
          如需创建 subIn 的实例对象
          必须先创建一个外部类 Out 的实例对象
          这是合理的
          因为 subIn 是非静态内部类 In 的子类
          非静态内部类 In 的实例对象里持有一个对外部类 Out 的实例对象的引用
          其子类 subIn 的实例对象里也应持有一个对外部类 Out 的实例对象的引用
          创建子类 subIn 的实例对象时传给 subIn 类构造器的 Out 对象
          就是子类 subIn 的实例对象里 Out 对象引用所指向的对象
          */
      }
}

/*
编译上面的程序将看到三个 *.class 字节码文件
HelloWorld.class
HelloWolrd$1In1.class
HelloWorld$1In2.class
局部内部类的 *.class 字节码文件总是遵循如下的命名格式：
OuterClass$NInnerClass.class
而成员内部类的 *.class 字节码文件总是遵循如下的命名格式： 
OuterClass$InnerClass.class
因为一个类里不可能有两个同名的成员内部类
但一个类里可以有多个同名的局部内部类（处于不同的方法中）
所以增加了一个 N 加以区分
*/
  • 局部内部类
    • 把一个内部类定义在方法里
    • 局部内部类只在该方法里有效，不能在外部类的方法以外的地方使用
    • 所以也不能用 static 修饰和使用访问控制符
    • 对于局部成员而言，不管是局部变量还是局部内部类
      • 它们的上一级程序单元都是方法，而不是类
      • 所以使用 static 修饰它们没有任何意义
      • 因此，所有的局部成员都不能使用 static 修饰
      • 且局部成员的作用域是所在的方法
      • 其他程序单元永远也不能访问另一个方法中的局部成员
      • 所以所有的局部成员都不能使用访问控制符
  • 局部内部类是一个非常“鸡肋”的语法
    • 在实际开发中很少定义局部内部类
    • 这是因为局部内部类的作用域太小了，只能在当前的方法中使用
    • 而大部分时候定义一个类，初衷是希望多次复用这个类

/*
对比以上两种方法
显然采用匿名内部类的写法更加简洁
*/
  • 匿名内部类
    • 适合创建那种只需使用一次的类
  • 创建匿名内部类时会立即创建一个该类的实例对象，这个类定义立即消失
    • 匿名内部类不能重复使用
  • 匿名内部类必须继承一个父类，或实现一个接口
    • 但最多只能继承一个父类，或实现一个接口
  • 匿名内部类不能是抽象类
    • 抽象类不能创建实例对象
    • 所以必须实现它的抽象父类或接口里包含的所有抽象方法
    • 也可以重写父类里的实例方法
  • 匿名内部类不能定义构造器
    • 没有类名
    • 但可以定义初始化块，可以通过初始化块来完成构造器需要完成的事情
    • 当通过实现接口来创建匿名内部类时，匿名内部类不能显式创建构造器，一次匿名内部类只有一个隐式的构造器，故 new 接口名后的括号内不能传入参数
    • 当通过继承父类了创建匿名内部类时，匿名内部类将拥有和父类相似的构造器，此处的相似指的是拥有相同的形参列表
  • 定义匿名内部类无须使用 class 关键字
  • 最常用的创建匿名内部类的方式是需要创建某个接口类型的实例对象
