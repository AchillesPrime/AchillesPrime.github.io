环境配置

  • JDK（Java SE Development Kit）：Java 标准版开发包
  • JRE（Java Runtime Environment）：Java 运行环境，包含于 JDK ，无需单独下载
  • JVM（Java Virtual Machine）： Java 虚拟机，包含于JRE，运行 Java 字节码文件，分为向上和向下两个部分，向上供给 Java 字节码文件完全相同的接口，向下适用不同平台的接口，实现 Java 程序的平台无关性
  • Oracle 于2009年收购了 Sun，所以请直接到 Oracle 官网下载：
    • Latest version JDK download
  • 安装配置过程：
    • 不建议安装在有空格的路径下，例如 C:\Program Files，可能会导致一些未知的问题
    • 我的电脑 —> 属性 —> 高级系统设置 —> 环境变量 —> 系统变量 —> 添加：
      1. JAVA_HOME：JDK安装路径
      2. Path：%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin
      3. CLASSPATH：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar
      4. 通常建议也添加一个 PATH 用户变量：%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin
      5. JDK1.5 以上的版本无需设置 CLASSPATH
      6. CLASSPATH 前面有个关键性的 . ，用以强制 Java 解释器在当前路径下搜索 Java 类
      7. 多一个空格或少一个空格，都可能引起错误哟

最简单的和最著名的程序

最简单的 Java 程序

class Test
{

}
  • 类（class）是 Java 最小的程序单位
    • 不允许可执行性语句、方法等成分独立存在
    • 所有程序部分都必须放在类定义里
    • 这里仅包含一个空类的定义

最著名的 Java 程序

public class HelloWorld
{
       public static void main(String[] args)
       {
              System.out.println ("Hello World!");
       }
}
  • mian 方法是 Java 程序的入口
    • main 方法的写法几乎是固定的
      • 必须使用 public 和 static 来修饰，这两个修饰符的位置可以互换
      • 必须使用 void 声明返回值
      • 必须是 (String[] args) 字符串数组类型的形参值
    • 对于那些不包含 main 方法的类
      • 也是有用的类
      • 一个大型的 Java 程序往往只需要一个 main 入口，即只有一个类包含 main 方法
      • 其他类都是用于被 main 方法直接或间接调用

运行

  • 将以上两个程序输出为 *.java 源文件
    • 源文件命名规则：
      • 扩展名必须是 .java
      • 一个 Java 源文件可以有多个类定义，但 public 类最多只能定义一个
      • 若包含 public 类定义，则源文件名必须与 public 类的类名相同
      • 两个良好的建议：
        • 一个 Java 源文件只定义一个类
        • 让源文件的主文件名与 public 类同名
  • 在源文件所在路径下的命令行窗口（cmd） 中编译、解释执行：
    • javac *.java
    • java Java类名
  • 空类可以被 javac 编译通过，但不可被 java 解释执行
    • 并不代表这个类是错的
    • java 解释器规定：
      • 如需某个类能被解释器直接解释执行，则这个类里必须包含 main 方法

注意要点

  • Java 严格区分大小写
    • HelloWorld 和 helloworld 是不一样的
      • 对的路只有一条，错误的路成千上万
    • class 和 Class 是不一样的
      • Java 中的关键字全部都是小写，无需任何大写
    • main 和 Main 是不一样的
      • Java 虚拟机只会从 main 方法开始执行
      • Main 方法只是个普通的方法，而不是程序的入口
  • 输出
    • System.out.print ()
      • 不会换行
    • System.out.println ()
      • 换行

GC（Garbage Collection）

  • 垃圾回收器
    • 后台自动释放内存区的垃圾（不再被引用的对象）
    • 整理出的内存分配给新的对象
  • 编写 Java 程序时的一个基本原则：
    • 对于不再需要的对象， 不要引用它们

面向对象简介

Java

  • 纯粹的面向对象编程语言，完全以对象为中心，完全支持面向对象的三种基本特征：
    • 封装：将对象的实现细节隐藏起来，并通过一些公用方法来暴露该类对外所提供的功能，从而提高了类的内聚性，降低了对象之间的耦合性
    • 继承：用已存在的类作为基础建立新的类，代码复用的重要手段，类之间共享属性和操作的机制，具有传递性
    • 多态：同一类型的对象执行同一方法，表现出不同的行为特征，继承是多态的基础，没有继承就没有多态

强调直接以现实世界中的对象为中心来思考 以主函数为入口，自顶向下，逐步求精，模块化

  • 基于对象：
    • 例如 JavaScript
    • 使用一些封装好的对象（实现了封装的特点）
    • 调用对象的方法，设置对象的属性
    • 但无法让开发者派生新的类（即没有继承的特点，更不可能有多态的特点）
    • 只能使用现有对象的方法和属性

一切都是对象

  • 在 Java 语言中，除了8个基本数据类型之外，一切都是对象
    • 对象就是面向对象程序设计的核心
    • 对象具有状态
      • 一个对象用数据值来描述它的状态
      • Java 通过为对象定义成员变量来描述对象的状态
    • 对象具有行为
      • 这些行为可以改变对象的状态
      • Java 通过为对象定义方法来描述对象的行为
    • 用 new 关键字创建指定类的对象
    • 对象实现了数据和行为的结合
      • 把数据和行为封装成了一个有机的整体
      • 提供了更大的编程粒度，更易于掌握和使用
    • Java 里的对象具有唯一性
      • 每个对象都有一个标识符来引用它
      • 失去标识符的对象将被垃圾回收器回收
      • Java 不允许直接访问对象，需通过对象的引用来操作对象

类与对象

  • 类是 Java 最小的程序单位
  • 不管多大的 Java 应用程序，都是由一个个的类组成
  • 类，是对具有相同或相似性质的一组对象的抽象
  • 对象，是类的实例，是类的具体存在
  • 一个类可以创建任意多个对象
  • 用 class 关键字定义类
    • 使用成员变量来描述该类对象的数据
    • 使用方法来描述该类对象的行为

总结

  • 成员变量 + 方法 = 类定义
  • 状态数据 + 行为 = 对象

类与类之间的结构关系

  • 一般 —> 特殊
    • 典型的继承关系
    • is a
    • 通过使用关键字 extends 来表示这种继承关系
    • 举例：
      • 水果 —> 苹果
      • 苹果 is a 水果
  • 整体 —> 部分
    • 典型的组合关系
    • has a
    • 通过在一个类里面保存另一个类的引用来实现这种组合关系

数据类型

分类

  • 基本数据类型（Primitive Type）
    • boolean 布尔型
    • 数值型
      • 整型
        • byte 、short 、int 、long
      • char 字符型
        • 实际上也是一种整数类型，相当于无符号整数类型
        • 字符串不是基本数据类型，字符串是一个类，也就是一个引用数据类型
      • 浮点型
        • float 、double
  • 引用数据类型（Reference Type）
    • 对一个对象的引用，对象包括实例和数组
    • 类、接口和数组类型
    • 特殊的空类型（null type）
      • 没有名称，因此不可能声明一个 null 类型的变量或者转换为 null 类型
      • null 引用是 null 类型表达式唯一可能的值
      • null 引用可以转换为任何引用类型
        • 且只能转换为引用类型，不能转化为基本类型
        • 无法对一个基本类型的变量赋予 null 值
      • 实际开发中，可以忽略 null 类型，假定 null 只是引用类型的一个特殊直接量
    • 实际上，引用类型变量就是一个指针，只不过 Java 语言里不再使用指针的说法

强类型语言

  • Java 是一门强类型（strongly typed）语言
    • 所有变量必须先声明、后使用
    • 指定类型的变量只能接受类型与之匹配的值
  • 强类型语言可以在编译过程中发现源代码的错误，保证程序的健壮性

注释、分隔符、标识符、关键字

  • 注释
    • 单行注释
      • // ......
    • 多行注释
      • /*......*/
    • 文档注释
      • /**......*/
  • 分隔符
    • ;
      • 语句的分隔
      • 一行可写多个语句，每个语句之间用分号隔开
      • 一个语句可跨多行，最后用分号结束
      • 字符串、变量名不能跨行
    • {}
      • 定义代码块
      • 成对出现
    • []
      • 访问数组元素
      • 紧跟数组变量名
      • 方括号内指定希望访问的数组元素的索引
    • （）
      • 形参声明
      • 实参传递
      • 表达式优先
      • 强制类型转换的运算符
      • Java 是一门格式自由的语言
      • 空格几乎可以出现在 Java 程序的任何地方
      • 可以出现任意多个空格
      • 但不要把一个变量名隔开成两个
      • Java 的空格包括空格符（Space）、制表符（Tab）和回车（Enter）等
    • .
      • 调用某个类或某个实例的指定成员
  • 标识符
    • 给程序中的变量、类、方法命名的符号
    • 开头：
      • 字母、下划线（_）、美元符（$）
    • Java 区分大小写
      • abc 和 Abc 是不同的标识符
    • 不允许使用单独的下划线 _ 作为标识符
      • 下划线 _ 必须与其他字符组合
    • 标识符不能是 Java 关键字和保留字
      • 但可以包含关键字和保留字
    • 标识符不能含空格
    • 标识符只能含美元符 $ ，不能含 @、# 等其他特殊字符
  • 关键字（keyword）
    • Java 所有的关键字（50个）都是小写
    • TRUE 、FALSE 、NULL 不是 Java 关键字
      • 这是 Java 的直接量（literal）
      • 标识符也不能使用这三个特殊的直接量
    • goto 、const 也成为保留字（reserved word）
      • Java 还未使用这两个关键字
      • 但可能在未来的 Java 版本中使用

基本数据类型—整型

  • byte
    • 1字节 = 8位
    • 表数范围：-128 ~ 127
    • [ -2^7] ~ [ 2^7 - 1]
  • short
    • 2字节 = 16位
    • 表数范围：-32768 ~ 32767
    • [ -2^15] ~ [ 2^15 - 1]
  • int
    • 4字节 = 32位
    • 表数范围：-2147483648 ~ 2147483647
    • [ -2^31] ~ [ 2^31 - 1]
  • long
    • 8字节 = 64位
    • 表数范围：
    • [ -2^63] ~ [ 2^63 - 1]
byte a1 = 18;
long a2 = 9999999999L;
long a3 = 9999;

System.out.println(a1);  //输出：18
System.out.println(a2);  //输出：9999999999
System.out.println(a3);  //输出：9999
  • int 是最常用的整数类型，直接给出一个整数类型默认就是 int 类型
    • 将一个较小的整数值（byte 或 short 表数范围之内）赋给 byte 或 short ，系统会自动把这个整数当成 byte 或 short 处理
    • 如果整数值超过 int 表数范围，且希望系统把它当作 long 类型来处理，需要在这个整数值后加上 L 作为后缀
    • 如果整数值在 int 表数范围之内，可以直接赋值给 long 类型的变量， int 类型会自动类型转换到 long 类型
  • 整数值表示方式
    • 二进制
      • 0b
      • 0B
    • 八进制
      • 0
    • 十进制
    • 十六进制
      • 0x
      • 0X
      • 10 ~ 15 = a ~ f = A ~ F
    • 二进制整数更加真实，更能表达整数在内存中的存在形式，在一些开发中使用二进制会更加便捷
int a4 = 0B 1000 0000 0000 0000 0000 0000 0000 0011; //32位

System.out.println(a4);//输出：-2147483645
  • 32位的二进制整数
    • 最高位32位为符号位，当符号位是1时，表明其为负数
    • 正如前面所指出的，整数值默认是 int 类型，若在二进制整数后添加 L ，这个二进制整数就会占64位，且第64位为符号位
    • 计算机以补码的形式保存所有的整数
      • 正数的原码和补码完全相同
      • 负数的转换情况如下所示：

  • 更多例子：

    byte a5 = (byte)0b 1110 1001;
/*
定义一个8位的二进制整数，数值默认为32位，因此最高位上无1，即为正数
强制转换为 byte 类型，产生溢出
最高位为1，为负数
1110 1001 ——> 1110 1000 ——> 1001 0111
*/
System.out.println(a5);//输出：-(2^4 + 2^2 + 2^1 + 2^0 )= -23

long a6 = 0b 1000 0000 0000 0000 0000 0000 0000 0011L;
/*
定义一个32位的二进制整数，数值默认为32位，符号位为1，即为负数
数值后添加 L 后缀，因此该整数实际占64位，符号位变为第64位，无1，即为正数
且有：原码 = 补码
*/
System.out.println(a6);//输出：2^32 + 2^1 + 2^0 = 2147483651

基本数据类型—字符型

  • Java 使用 Unicode 字符集作为编码方式
    • 2字节 =16位
    • Unicode 被设计成支持世界上所有书面语言的字符集
    • 因此 Java 支持各种语言的字符
  • 字符型值必须使用英文单引号 '' 括起来
  char b1 = 'a';       //单个字符
  char b2 = '\'';      //转义字符 
  char b3 = '\"';      //转义字符
  char b4 = '帅';    //单个字符
  char b5 = '\u9999';  //Unicode 值，'\u9999'对应'香'字
  char b6 = 107;       //编号107对应字符 k 
  int  b7 = 'k';       //k 的 ASCII 码值为107


  System.out.println(b1);  //输出： a
  System.out.println(b2);  //输出： '
  System.out.println(b3);  //输出： "
  System.out.println(b4);  //输出： 帅
  System.out.println(b5);  //输出： 香
  System.out.println(b6);  //输出： k
  System.out.println(b7);  //输出： 107
  • 字符型值的三种表示形式
    • 单个字符
      • ‘A’ 、’9’ 、’0’
    • 转义字符
      • ‘\n’ 、’\t’
    • 直接使用 Unicode 值表示字符
      • ‘\uXXXX’
      • 其中 XXXX 表示一个十六进制的整数，范围为 ‘\u0000’ ~ ‘\uFFFF’ ，共65536个
      • 其中 ‘\u0000’ ~ ‘\u00FF’ 即为 ASCII 码值，共256个
  • 计算机底层保存字符时，实际上是保存字符的编号，因此 char 类型也可以直接作为整型值来使用，它相当于一个16位的无符号整数，表数范围：0 ~ 65535 ，共65536个
  • char 类型的变量、值参与加、减、乘、除等运算，实际上便是该字符对应的编码参与运算
  • 如果把 0 ~ 65535 范围内的 int 整数赋给 char 类型变量，系统会自动把这个 int 整数当成 char 类型来处理
String s1 = "世间安得双全法，不负如来不负卿";  //字符串
String s2 = "C:\\Java";                   //转义字符

System.out.println(s1);  //输出：世间安得双全法，不负如来不负卿
System.out.println(s2); //输出：C：\Java
  • Java 没有提供字符串的基本数据类型，而是通过 String 类来表示字符串
  • 字符串要使用英文双引号 "" 括起来
  • Java 中的 ' 、" 、\ 都有特殊用途
  • 如果一个字符串中包含了这些特殊字符，则应该用转义字符辅助表示
    • \b——退格符
    • \n——换行符
    • \r——回车符
    • \t——制表符
    • \"——双引号
    • \'——单引号
    • \\——反斜线
    • Java 会把第一个反斜线 \ 当成转义字符

基本数据类型—浮点型

  • Java 的浮点类型有固定的表数范围和字段长度，遵循 IEEE 754 标准，与机器无关
  • float
    • 单精度浮点数
    • 4字节 = 32位
  • double
    • 双精度浮点数

  • 表示形式
    • 十进制
      • 5.12 、512.0 、.512

    • 科学计数法
      • 5.12e2 = 5.12 * 10^2
      • 5.12E2 = 5.12 * 10^2
      • （e 或 E 之前的数）* 10^（e 或 E 之后的数）

  • Java 的浮点类型默认是 double 类型
    • 5.12 表示 double 类型，占64位内存空间
    • 5.12f 或 5.12F 表示 float 类型 ，占32位内存空间
    • 在数值后面加 d 或 D 后缀，强制指定为 double 类型，但通常没必要
double c1 = 0.0;
float  c2 = Float.POSITIVE_INFINITY;     //float 型正无穷大

System.out.println(-8 / c1);                 //输出：-Infinity
System.out.println(-8 / 0);                  //输出：ArithmeticException：/by zero
  • 三个特殊的浮点数值，用于表示溢出或出错
    • 正无穷大
      • 用一个正数除以0得到
      • 所有正无穷大数值都是相等的
    • 负无穷大
      • 用一个负数除以0得到
      • 所有负无穷大数值都是相等的
    • 非数
      • 0.0除以0.0 、对负数开方得到
      • 一个非数不与任何数值相等，也不与任何非数相等
    • 只有浮点数除以0才能得到正无穷大或负无穷大
    • Java 会自动把和浮点数运算的0（整数）当作0.0（浮点数）处理
    • 一个整数除以0，将抛出一个异常：ArithmeticException：/by zero（除以0异常）

    • float

  • 只有一个 boolean 类型
  • 用于表示逻辑上的“真”和“假”
  • boolean 类型的数值只能是 true 或 false
  • 不能用0或非0代替
  • 其他基本数据类型的值也不能转换成 boolean 类型
  • 只需1位即可保存，但计算机允许分配的最小内存单元是1字节（8位），因此实际上占用8位
  • 使用 boolean 类型的值和字符串进行连接运算， boolean 类型的值会自动转换成字符串

直接量

  • 程序中通过源代码直接给出的值
    • int 类型直接量
      • 在程序中直接给出的整型数值
      • 二进制 、八进制 、十进制 、十六进制
    • long 类型直接量
      • 在整型数值后加上 L 后缀（也可以加 1 ，但是容易混淆）
    • char 类型直接量
      • 单引号括起来的字符
      • 转义字符
      • Unicode 值
    • double 类型直接量
      • 标准小数形式 、科学计数法
    • float 类型直接量
      • 在一个浮点型数值后加上 f 或 F 后缀
      • 标准小数形式 、科学计数法
    • boolean 类型直接量
      • true
      • false
    • String 类型直接量
      • 双引号括起来的字符序列
    • null 类型直接量
      • null
  • boolean 类型的直接量只能赋值给 boolean 类型
  • null 类型的直接量可以直接赋给任何引用类型
