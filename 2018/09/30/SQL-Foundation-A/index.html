简介

  • SQL Server 是美国 Microsoft 公司推出的关系型数据库管理系统（RDBMS）
  • Oracle 是美国 Oracle 公司推出的关系型数据库管理系统
  • MySQL 关系型数据库管理系统，由瑞典 MySQL AB 公司开发，目前属于 Oracle 旗下产品
  • Oracle 11g
    • 帮助企业管理企业信息
    • 更深入地洞察业务状况并做出调整
    • 适应不断变化的竞争环境

范式

  • 关系型数据库中要满足的一定要求

  • 规范化有助于消除表中数据的冗余和不一致性

  • 为了充分发挥 SQL 的作用，往往是转化到第三范式

  • 规范化的一个简单原则：所有的属性必须完全依赖于主关键字列

  • 第一范式（1NF）

    • 所有属性都是不可分割的

    • 表中的每一行每一列均有一个值，永远不会是一组值

  • 第二范式（2NF）

    • 每一个不是关键字一部分的列都依赖于关键字

    • 如果函数仅依赖于一部分主码（主码有若干个）来确定信息，则违反了第二范式

  • 第三范式 （3NF）

    • 每个非关键字列都独立于其他非关键字列，并依赖于关键字

    • 违反第三范式经常产生严重后果，必须予以纠正

SQL、PL/SQL、SQL*PLUS

  • SQL（结构化查询语言）是世界上最流行的、标准化的数据库语言
    • 4GL ，第四代语言
    • 用于建立、存储、修改、检索和管理关系型数据库中的数据
    • 非过程化程度相当高，用户只需在程序中指出要做什么就可以了
    • 在 BASIC、C++、Java 等语言中
      • 3GL ，第三代语言
      • 需要把对数据进行处理的每一过程都在程序中表达清楚
      • 否则程序就不能正确反映用户的要求与意图
      • 这样的语言称之为过程化语言
  • PL/SQL 是一种过程化语言，是对SQL的扩充
    • 3GL ，第三代语言
    • 把SQL语言的特点和第三代语言（3GL）的强大功能结合在一起
    • 可以用来实现比较复杂的业务逻辑，成为设计复杂数据库应用程序的有力工具
  • SQL*Plus 是 Oracle 数据库的开发工具
    • 在 SQL*Plus 环境中可以编辑、调试、运行 SQL 语言语句和 PL/SQL 程序
    • 对英文字母大小写不敏感
    • 增加程序的可读性：
      • 关键字大写
      • 标识符小写
    • SQL> 是SQL*Plus的提示符
    • 退出操作
      • SQL>exit（出口、通道）
      • SQL>quit（辞职、中断）
    • 在SQL*Plus 中执行 SQL 语句，语句末尾需要加上分号 “；”
    • 在SQL*Plus 中执行 PL/SQL 程序，先需执行 “set serveroutput on”
      • 且最后一条语句的下一行需要输入斜杠 “/“

基本数据类型

  • Oracle 11g 主要的3种数据类型：基本、集合、引用。

  • char：定长字符串，必须指定长度，1（默认）~255字节

  • varchar2：变长字符串，必须指定最大字节数，最大长度为2000字节

  • number：存储零、正负定点数或浮点数，最大精度38位

    • 定点数据类型number（p,s）

    • p：总的有效数字的位数，精度范围1~38

    • s：小数点右边数字位，范围-84~127

日期格式

select sysdate from dual;
select * from dual;
select user from dual;
select 1+2 from dual;
select to_char(sysdate,'yyyy-mm-dd hh24:mi:ss') from dual;

  • select sysdate from dual

    • 显示 Oracle 系统日期

  • sysdate 为 Oracle 系统当前服务器的时间

    • dual 是一个单行单列的虚拟表

      • dual 表数据类型为 VERCHAR2(1)

      • Oracle 的内部逻辑保证 dual 表中永远只有一条数据 “X”

      • dual 可用来选择系统变量或求一个表达式的值

select sysdate , to_char (sysdate , 'yyyy-mm-dd') from dual;
select sysdate , to_char (sysdate , 'yyyy/mm/dd') from dual;
select sysdate , to_char (sysdate , 'yyyymmdd') from dual;
select sysdate , to_char (sysdate , 'yyyymmdd hh24:mi:ss') from dual;
select sysdate , to_char (sysdate , 'yyyy') from dual;
select sysdate , to_char (sysdate , 'mm') from dual;
select sysdate , to_char (sysdate , 'hh24') from dual;
select sysdate , to_char (sysdate , 'mi') from dual;
  • to_char ()函数
    • 将时间日期按照指定的格式输出，得到的是字符串，而非date类型
select sysdate , to_date ('20170615' , 'yyyy-mm-dd') from dual;
select sysdate , to_date ('20170615' , 'yyyy/mm/dd') from dual;
select sysdate , to_date ('20170615' , 'yyyymmdd') from dual;
select sysdate , to_date ('20170615' , 'yyyy-mm-dd hh24:mi:ss') from dual;
  • to_date()函数
    • 将字符串转换为具体指定的时间日期格式
  • 如果安装的客户端是中文环境，格式 mon 就不能用英文的月份写法
    • 必须用中文的 “数字-数字+月-数字”
    • 或者使用：
      • to_date(‘1998-05-07’,’yyyy-mm-dd’)

建表

create table students 
(
       student_id number(5)
                  constraint student_pk primary key,
       monitor_id number(5),
       name varchar2(10) 
            not null,
       sex varchar2(6)
           constraint sex_chk check(sex in ('男','女')),
       dob date,
       specialty varchar2(10)   
);

create table departments 
(
       department_id number(3)
                     constraint department_pk primary key,
       department_name varchar2(8) 
                       not null,
       address varchar2(40)
);

create table teachers 
(
       teacher_id number(5)
                  constraint teacher_pk primary key,
       name varchar2(8) 
            not null,
       title varchar2(6),
       hire_date date 
                 default sysdate,
       bonus number(7,2),
       wage number(7,2),
       department_id number(3)
                     constraint teachers_fk_departments
                     references departments(department_id)
);


create table courses
(
       course_id number(5)
                 constraint course_pk primary key,
       course_name varchar2(30) 
                   not null,
       credit_hour number(2)
);


create table student_grade
(
       student_id number(5)
                  constraint student_grade_fk_students
                  references students(student_id),
       course_id number(5)
                 constraint student_grade_fk_courses
                 references courses(course_id),
       score number(4,1)
);

create table grades
(
       garde_id number(1)
                constraint grade_ok primary key,
       low_score number(4 , 1),
       high_score number(4 , 1),
       grade varchar2(6)
);

create table minors
(
       minor_id number(5)
                constraint minor_pk primary key,
       minor_name varchar2(30) 
                  not null,
       credit_hour number(2)
);

create table courses2
(
       course_id number(5)
                 constraint course2_pk primary key,
       course_name varchar2(30) 
                   not null,
       credit_hour number(2)  
);

create table student_computer
(
       student_id number(5)
                  constraint student_computer_pk primary key,
       monitor_id number(5),
       name varchar2(10) 
            not null,
       sex varchar2(6),
       dob date,
       specialty varchar2(10)
);
