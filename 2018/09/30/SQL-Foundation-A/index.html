范式

  • 关系型数据库中要满足的一定要求
  • 规范化有助于消除表中数据的冗余和不一致性
  • 为了充分发挥 SQL 的作用，往往是转化到第三范式
  • 规范化的一个简单原则：所有的属性必须完全依赖于主关键字列
  • 第一范式（1NF）
    • 所有属性都是不可分割的
    • 表中的每一行每一列均有一个值，永远不会是一组值
  • 第二范式（2NF）
    • 每一个不是关键字一部分的列都依赖于关键字
    • 如果函数仅依赖于一部分主码（主码有若干个）来确定信息，则违反了第二范式
  • 第三范式 （3NF）
    • 每个非关键字列都独立于其他非关键字列，并依赖于关键字
    • 违反第三范式经常产生严重后果，必须予以纠正

SQL、PL/SQL、SQL*PLUS

  • SQL（结构化查询语言）是世界上最流行的、标准化的数据库语言
    • 4GL ，第四代语言
    • 用于建立、存储、修改、检索和管理关系型数据库中的数据
    • 非过程化程度相当高，用户只需在程序中指出要做什么就可以了
    • 在 BASIC、C++、Java 等语言中
      • 3GL ，第三代语言
      • 需要把对数据进行处理的每一过程都在程序中表达清楚
      • 否则程序就不能正确反映用户的要求与意图
      • 这样的语言称之为过程化语言
  • PL/SQL 是一种过程化语言，是对SQL的扩充
    • 3GL ，第三代语言
    • 把SQL语言的特点和第三代语言（3GL）的强大功能结合在一起
    • 可以用来实现比较复杂的业务逻辑，成为设计复杂数据库应用程序的有力工具
  • SQL*Plus 是 Oracle 数据库的开发工具
    • 在 SQL*Plus 环境中可以编辑、调试、运行 SQL 语言语句和 PL/SQL 程序
    • 对英文字母大小写不敏感
    • 增加程序的可读性：
      • 关键字大写
      • 标识符小写
    • SQL> 是SQL*Plus的提示符
    • 退出操作
      • SQL>exit（出口、通道）
      • SQL>quit（辞职、中断）
    • 在SQL*Plus 中执行 SQL 语句，语句末尾需要加上分号 ；
    • 在SQL*Plus 中执行 PL/SQL 程序，先需执行 set serveroutput on
      • 且最后一条语句的下一行需要输入斜杠 /

基本数据类型

  • Oracle 11g 主要的3种数据类型：基本、集合、引用。
  • char：定长字符串，必须指定长度，1（默认）~255字节
  • varchar2：变长字符串，必须指定最大字节数，最大长度为2000字节
  • number：存储零、正负定点数或浮点数，最大精度38位
    • 定点数据类型number（p,s）
    • p：总的有效数字的位数，精度范围1~38
    • s：小数点右边数字位，范围-84~127

定义表结构

create table students 
(
       student_id number(5)
                  constraint student_pk primary key,
       monitor_id number(5),
       name varchar2(10) 
            not null,
       sex varchar2(6)
           constraint sex_chk check(sex in ('男','女')),
       dob date,
       specialty varchar2(10)   
);

create table departments 
(
       department_id number(3)
                     constraint department_pk primary key,
       department_name varchar2(8) 
                       not null,
       address varchar2(40)
);

create table teachers 
(
       teacher_id number(5)
                  constraint teacher_pk primary key,
       name varchar2(8) 
            not null,
       title varchar2(6),
       hire_date date 
                 default sysdate,
       bonus number(7,2),
       wage number(7,2),
       department_id number(3)
                     constraint teachers_fk_departments
                     references departments(department_id)
);


create table courses
(
       course_id number(5)
                 constraint course_pk primary key,
       course_name varchar2(30) 
                   not null,
       credit_hour number(2)
);


create table student_grade
(
       student_id number(5)
                  constraint student_grade_fk_students
                  references students(student_id),
       course_id number(5)
                 constraint student_grade_fk_courses
                 references courses(course_id),
       score number(4,1)
);

create table grades
(
       garde_id number(1)
                constraint grade_ok primary key,
       low_score number(4 , 1),
       high_score number(4 , 1),
       grade varchar2(6)
);

create table minors
(
       minor_id number(5)
                constraint minor_pk primary key,
       minor_name varchar2(30) 
                  not null,
       credit_hour number(2)
);

create table courses2
(
       course_id number(5)
                 constraint course2_pk primary key,
       course_name varchar2(30) 
                   not null,
       credit_hour number(2)  
);

create table student_computer
(
       student_id number(5)
                  constraint student_computer_pk primary key,
       monitor_id number(5),
       name varchar2(10) 
            not null,
       sex varchar2(6),
       dob date,
       specialty varchar2(10)
);

数据操纵语言（DML）

  • insert(增)
  • delete(删)
  • uodate(改)
  • 以事务为单位，一个事务为一个工作的逻辑单位，是一个 SQL 语句序列
  • 执行每一个 DML 语句，所有的操作都在内存中完成
  • 执行完一系列 DML 语句后，需要决定由DML语句所做的修改是
    • 全部或部分地保留到硬盘文件上
    • 还是全部撤销
    • 这些决定由事务处理语句来完成
      • commit（提交）
      • savepoint（保留点）
      • rollback（撤销）
        • 事务被用来确保数据库中数据的一致性
        • 要么全部成功
        • 要么全部取消
    • 数据库事务开始于应用程序中的第一条 DLM 语句
    • 数据库事务开始后：
      • Oracle 会为 DLM 操作涉及的表加上表锁
        • 防止其他用户改变表结构
      • Oracle 会为 DLM 操作涉及的行加上行锁
        • 防止其他事务在相应的行上执行 DML 操作
    • 数据库事务结束后，释放封锁（表锁和行锁）
  • insert
    • 数字列，直接提供数值
    • 字符列，''英文单引号
    • 日期列，''英文单引号
    • 提供数据的顺序问题
      • 或与表中列的顺序完全一致
      • 或与提供的列表中的列名顺序一致
    • 必须为主键和 not null 提供数据
      • 未指定列的值
        • 或 null（可显示处理）
        • 或 default（可显示处理）

数据库的完整性

  • 实体完整性：
    • 表的主键不能为空值 null
    • 表的主键不能取重复值

  • 参照完整性

    • teachers 表
    • 通过 department_id 列与
    • departments 表
    • 建立了参照完整性
      • 从表 teachers
      • 主表 departments
    • 从表 teachers 中的 department_id 只能取
      • 或 null 值
      • 或 departments 主表中的 department_id 值

  • 违反参照完整性约束规则：

    • 1、

      delete from departments
       where 
       department_id = 101;
      • 主表要删除的 department_id ，从表中存在

    • 2、

      update departments set 
       department_id = 105 
       where 
       department_id = 102;
      • 主表要修改的 department_id ，从表中存在

    • 3、

      insert into teachers 
values (......, 108);
      • 从表中插入的 department_id ，主表中无

    • 4、

      update teachers set 
       department_id = 107
       where 
       teacher_id = 10106;
      • 从表要修改的 department_id ，主表中无

  • 自定义完整性

    • 1、

      • 例如插入学生记录时，没有给出姓名值
      • 违反了 not null 自定义完整性约束

    • 2、

      update student set
       sex = '南' 
       where 
       student_id = 10205;
      • 违反了 check 自定义完整性约束（只能取‘男’或‘女’）

事务处理

  • commit
    • （显示处理事务）提交事务
    • 结束当前事务
    • 把当前事务所执行的全部修改保存到外存的数据库中
    • 删除事务的所有保留点
    • 释放该事务的封锁
  • rollback
    • （显示处理事务）撤销事务
    • 结束当前事务
    • 撤销当前事务中所做的全部修改
    • 删除当前事务中所建的全部保留点
    • 释放该事务的封锁
update departments set
       address = '5号教学楼'
       where 
       department_id = 104;
savepoint sp1;
delete from departments 
       where 
       department_id = 104;
rollback to sp1;
  • savepoint
    • rollback to sp1
      • 撤销保留点 sp1 之后的全部事务
      • 删除保留点 sp1 之后的全部保留点
  • 隐式处理事务：
    • 执行 create table 、alter table 、drop table 等语句时
      • Oracle 系统自动提交 commit 事务
    • 执行 exit（正常）退出 SQL*Plus 时
      • Oracle 系统自动提交 commit 事务
    • 当意外（非正常）退出 SQL*Plus 时
      • Oracle 系统自动撤销 rollback 事务
  • set transaction read only
    • （特殊事务）只读事务
    • 只允许查询语句
    • 不允许执行 DML 语句
    • 可以取得特定时间点的数据
    • 该命令必须为事务的第一条语句
    • 必须由 commit 、rollback 语句来结束只读事务

    • 使用括号可以改变运算次序

select name as "姓名" , nvl2(bonus , bonus+wage , wage) as "月总收入" 
       from teachers;
select name as "姓名" , coalesce(bonus+wage , wage) as "月总收入" 
       from teachers;
  • 任何数据类型的列
    • 只要没有使用非空 not null 或主键 primary key 完整性约束
    • 就都有可能出现空值 null
    • 空值 null 没有数据类型
    • 空值 null 出现在字符串连接表达式中，Oracle 将其作为空串处理
    • 空值 null 出现在算术表达式中，运算结果为 null
    • 为了解决空值 null 参加运算时出现的问题，Oracle 提供了3个函数对此进行处理
    • nvl（A , B）
      ​ - A 、B 的数据类型必须匹配
      ​ - A 为 null ，返回 B
      ​ - A 不为 null ，返回 A
    • nvl2（A ，B ，C）
      ​ - A 、B 、C 的数据类型必须匹配
      ​ - A 为 null ,返回 C
      ​ - A 不为 null ，返回 B
    • coalesce（A、B、C、D…）
      ​ - 联合，合并
      ​ - 返回参数列表中的第一个非空值
      ​ - 若参数列表皆为空值，最终返回一个空值 null
