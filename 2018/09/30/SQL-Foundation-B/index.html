子查询

  • 子查询
    • 嵌入在其他 SQL 语句中的 select 语句
    • 本质
      • 多个select语句的嵌套

单行子查询

  • 返回一行一列给外部（主）SQL 语句
select * from teachers
       where 
       wage < (select avg(wage) from teachers);
select * from students
       where 
       specialty = (select specialty from students 
                           where 
                           name = '王天仪');
select * from students
       where 
       dob > (select dob from students 
                     where 
                     name = '王天仪');
  • where
    • 子查询的结果作为主查询的查询条件
select department_id , avg(wage) as "平均工资" from teachers
       group by department_id
       having 
       avg(wage) > (select min(avg(wage)) from teachers 
                           group by department_id);
  • having
    • 子查询的结果作为主查询的分组条件
select * from (select * from students 
                      where 
                      sex = '男')
       where 
       specialty = '计算机';
  • from
    • 子查询的结果作为主查询的视图

多行子查询

  • 返回多行一列给外部（主）SQL 语句
  • where 子句中必须使用多行比较符 in、any或all
select student_id , name from students
       where 
       student_id in 
       (select student_id from students 
               where 
               name like '王%');
select course_id , course_name from courses
       where 
       course_id not in (select course_id from student_grade);
  • in
    • 匹配于子查询的任意一个值
    • 结果为真，否则为假
select * from teachers
       where 
       wage < any
       (select avg(wage) from teachers 
               group by department_id);

  • any

    • 只要符合子查询结果的任意一个值
    • 结果为真，否则为假

    • = any

      • 功能与IN操作符是完全一样的

    • > any

      • 比子查询中返回的最小结果还要大

    • < any

      • 比子查询中返回的最大结果要小
select * from teachers
       where 
       wage > all 
       (select avg(wage) from teachers 
               group by department_id);
  • all
    • 必须符合子查询结果的所有值
    • 结果为真，否则为假
    • > all
      • 比子查询中返回的最大结果还要大
    • < all
      • 比子查询中返回的最小结果还要小

多列子查询

  • 返回一行多列或多行多列给外部（主）SQL 语句
select * from students
       where 
       (specialty , dob) =
       (select specialty , dob from students 
               where 
               name = '王天仪');
  • 返回一行多列
    • 参照单行子查询
select * from teachers
       where 
       (department_id , wage) in 
       (select department_id , min(wage) from teachers 
               group by department_id);
  • 返回多行多列
    • 参照多行子查询

相关子查询

  • 子查询引用了外部（主）查询中的一列或多列
    • 子查询不能在外部（主）查询之前求值
    • 需要依靠外部查询才能获得值
    • 执行过程：
      • 1、
        • 从外层查询中取出一个元组，将元组相关列的值传给内层查询
      • 2、
        • 执行内层查询，得到子查询操作的值
      • 3、
        • 外查询根据子查询返回的结果或结果集得到满足条件的行
      • 4、
        • 然后外层查询取出下一个元组重复做步骤1-3，直到外层的元组全部处理完毕
select * from teachers t1
       where 
       wage > (select avg(wage) from teachers t2
                      where 
                      t2.department_id = t1.department_id);
  • 外部（主）查询和子查询通过 department_id 相关联
    • 外部（主）查询从 teacher 表中检索出所有行
    • 并传递给子查询
    • 子查询接受外部（主）查询传递过来的每一行数据
    • 并对满足 t2.department_id = t1.department_id 的每一部门教师计算平均工资
select course_id , course_name from courses c
       where 
       exists (select 2 from student_grade sg
                      where 
                      sg.course_id = c.course_id);
  • exists
  • 已被选学的课程
    • 用于检查子查询返回的记录行是否存在
    • 分析执行步骤：
      • 1、
        • 先执行外部查询
        • SQL> select course_id ,course_name from courses c
      • 2、
        • 外部查询的每一条数据对比一次子查询
        • 第一次比较
          • 10101 计算机组成原理
          • 只要 exists 后对比的结果返回true
          • 则列出 10101 计算机组成原理
      • 3、
        • 同理外部查询的 course_id , course_name 还剩下8项
        • 一一和内部查询的 student_id , course_id , grade 比较
        • 最后得出结果
        • 不管第二个 select 用 * 号还是 student_id , course_id , grade 项
        • 都不会影响整个相关查询的结果，
        • 外部查询的项和内部查询项对比时只会看是否为ture，如果为ture则列出结果
        • 用 select 2 不用 select *
        • 因为用 select * 要查数据字典所以会慢点
select course_id , course_name from courses c
       where 
       not exists(select 2 from student_grade sg
                         where 
                         sg.course_id = c.course_id);
  • not exists
  • 未被选学的课程
    • 用于检查子查询返回的记录行是否不存在
select department_id , department_name from departments
       where 
       department_id in (select department_id from teachers);
  • in
    • 已经安排教师的系部
select department_id , department_name from departments
       where 
       department_id not in (select department_id from teachers);
  • not in
    • 没有安排教师的系部

嵌套子查询

select * from (select * from students 
                      where 
                      specialty = (select specialty from students 
                                          where 
                                          name = '王天仪')
               );
  • 嵌套深度允许高达255级

集合操作

select course_id , course_name , credit_hour from courses
       union all
select minor_id , minor_name , credit_hour from minors;
select course_id , course_name , credit_hour from courses
       union all
select minor_id , minor_name , credit_hour from minors
       order by 1;
select * from students 
       where 
       specialty = '计算机'
       union all
select * from students 
       where 
       sex = '男';
select * from students 
       where 
       specialty = '计算机'
       union all
select * from students 
       where 
       sex = '男'
       order by 1;
  • union all
    • 获取两个查询结果的并集
    • 不取消重复行
    • 不排序
select course_id , course_name , credit_hour from courses
       union
select minor_id , minor_name , credit_hour from minors;
select * from students 
       where 
       specialty = '计算机'
       union
select * from students 
       where 
       sex = '男';
  • union
    • 获取两个查询结果的并集
    • 自动取消重复行
    • 以第一列排序
select course_id , course_name , credit_hour from courses
       intersect
select minor_id , minor_name , credit_hour from minors;
select * from students 
       where 
       specialty = '计算机'
       intersect
select * from students 
       where 
       sex = '男';
  • intersect
    • 获取两个查询结果的交集
    • 第一个查询结果集中存在
    • 第二个查询结果集中存在
    • 以第一列排序
select course_id , course_name , credit_hour from courses
       minus
select minor_id , minor_name , credit_hour from minors;
select * from students 
       where 
       specialty = '计算机'
       minus
select * from students 
       where 
       sex = '男';
  • minus
    • 获取两个集合的差集
    • 第一个查询结果集中存在
    • 第二个查询结果集不存在
    • 以第一列排序
(select course_id , course_name , credit_hour from courses
       intersect
select minor_id , minor_name , credit_hour from minors)
       union
select course_id , course_name , credit_hour from courses2;
select course_id , course_name , credit_hour from courses
       union
(select minor_id , minor_name , credit_hour from minors
       intersect
select course_id , course_name , credit_hour from courses2);
  • 组合使用集合操作符
    • 具有相同的优先级
    • 从左至右
    • 括号内的集合操作具有较高的优先级
select course_id , course_name , credit_hour from courses
       union
select minor_id , minor_name , credit_hour from minors
       order by course_name;
select course_id , course_name as name , credit_hour from courses
       union
select minor_id , minor_name , credit_hour from minors
       order by name;
select course_id , course_name , credit_hour from courses
       union
select minor_id , minor_name , credit_hour from minors
       order by 2;
  • 集合操作符只能有一个 order by 子句
    • 必须将它放在语句的末尾
    • 排序的3种较好的选择
      • 第一个 select 中的列名
      • 第一个 select 中的列别名
      • 集合操作结果集中的位置编号
select course_name as "111", sum(credit_hour) from courses
       where 
       credit_hour > 3
       group by course_name
       union
select minor_name , sum(credit_hour) from minors
       where 
       credit_hour > 2
       group by minor_name;
  • 为某个列指定一个新的名称（列别名），必须在集合操作的第一个select子句中指定

集合操作中的数据类型

create table table_1 
(
       column_11 number(3),
       column_12 varchar2(2)
);


create table table_2 (
       column_21 number(5),
       column_22 varchar2(4)
);


insert into table_1
values (111 , 'aa');
insert into table_1
values (222 , 'bb');
insert into table_1
values (333 , 'cc');


insert into table_2
values (44444 , 'dddd');
insert into table_2
values (55555 , 'eeee');
insert into table_2
values (66666 , 'ffff');


select * from table_1;
select * from table_2;
delet from table_1;
delet from table_2;
drop table table_1;
drop table table_2;

select column_11 , column_12 from table_1
       union
select column_21 , column_22 from table_2;
  • 集合操作中的数据类型相同，长度不同
    • 文本
      • 让所有数据的长度等于最长列的长度
    • 数值
      • 让所有数据的长度保持所有数值都允许的长度
    • 日期
      • 日期列的数据类型只有一个，不存在数据类型相同但长度不同的情况
create table table_3 
(
       column_31 number(3),
       column_32 varchar2(2),
       column_33 varchar2(10)
);


create table table_4 
(
       column_41 varchar2(3),
       column_42 number(2),
       column_43 date
);


insert into table_3
values (111 ,'aa' , 'aaaaaaaaaa');
insert into table_3
values (222 ,'bb' , 'bbbbbbbbbb');
insert into table_3
values (333 ,'cc' , 'cccccccccc');


insert into table_4
values ('ddd' ,44 , '07-5月-1998');
insert into table_4
values ('eee' ,55 , '07-5月-1998');
insert into table_4
values ('fff' ,66 , '07-5月-1998');


select * from table_3;
select * from table_4;
delet from table_3;
delet from table_4;
drop table table_3;
drop table table_4;


select to_char (column_31) , 
       column_32 , 
       column_33 from table_3
       union
select column_41 , 
       to_char (column_42) , 
       to_char (column_43 , 'yyyy-mm-dd') from table_4;
  • 集合操作中的数据类型不同
    • 通过 to_char() 函数将所有数字型或日期型转换为文本型
      • 查询前的设置
        • SQL> column column_31 format a10;
          • 字段column_31输出时的格式设置为文本型，且占10个字符宽度
        • SQL> column column_32 format a10;
          • 字段column_32输出时的格式设置为文本型，且占10个字符宽度
        • SQL> column column_33 format a10;
          • 字段column_33输出时的格式设置为文本型，且占10个字符宽度
